# ADR-0001: 운영자 JWT를 httpOnly 쿠키로 저장

## 상태 (Status)

Accepted - 2026-06-02

## 컨텍스트 (Context)

운영자(User 1인)는 로그인 후 Post 작성·수정·삭제·발행 등 쓰기 작업을 수행한다. 인증 토큰(JWT)을 프론트엔드에서 어디에 보관할지 결정해야 한다. PRD 비기능 요구 NF1(보안 - 인증 정보 보호)에 따라 XSS로 인한 토큰 탈취를 막아야 한다. 독자는 인증이 필요 없어 인증 표면은 운영자 화면에 한정된다.

평가 기준: XSS 안전성, CSRF 노출, 구현 복잡도.

## 결정 (Decision)

JWT를 `httpOnly` + `Secure` + `SameSite=Lax` 쿠키(`access_token`)에 저장한다. 토큰은 JavaScript에서 접근할 수 없으며, axios 요청 시 자동 동봉된다.

## 결과 (Consequences)

긍정:
- XSS가 발생해도 JS에서 토큰을 읽을 수 없어 탈취 위험이 낮다.
- 프론트에서 토큰 저장/주입 로직이 불필요하다.

부정/감수해야 할 것:
- CSRF 표면이 생긴다 → SameSite=Lax로 1차 방어, 필요 시 CSRF 토큰 추가 검토.
- 쿠키 기반이라 순수 토큰 헤더 방식보다 CORS·도메인 설정에 주의가 필요하다.

## 검토 시점

다중 클라이언트(모바일 앱 등)나 서드파티 API 소비자가 생겨 헤더 기반 토큰이 필요해질 때 재평가한다.

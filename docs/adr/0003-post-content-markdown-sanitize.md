# ADR-0003: Post 본문을 마크다운으로 저장하고 렌더 시 새니타이즈

## 상태 (Status)

Accepted - 2026-06-02

## 컨텍스트 (Context)

Post 본문의 저장·표현 형식을 정해야 한다. PRD에서 이미지·미디어 업로드는 범위 외이며 본문은 텍스트/마크다운으로 한정된다. 동시에 NF1(보안 - XSS)에 따라 렌더링 시 악성 스크립트가 실행되지 않아야 한다.

평가 기준: 표현력, XSS 안전성, 구현 복잡도.

## 결정 (Decision)

Post 본문은 마크다운 원문(`contentMarkdown`)으로 DB에 저장하고, 프론트에서 `react-markdown` + `rehype-sanitize`로 렌더링하여 위험한 태그·속성을 제거한다.

## 결과 (Consequences)

긍정:
- 제목·목록·링크 등 충분한 표현력을 제공한다.
- 원문을 보존하므로 추후 다른 렌더러로 전환·재처리가 쉽다.
- sanitize로 XSS를 차단한다.

부정/감수해야 할 것:
- 렌더 시 새니타이즈 비용과 렌더러 의존성이 추가된다.
- 마크다운 외 서식(임베드 등)은 추가 작업이 필요하다.

## 검토 시점

리치 에디터·이미지 업로드 등 본문 기능이 확장되거나, 서버 측 렌더링/캐싱이 필요해질 때 재평가한다.

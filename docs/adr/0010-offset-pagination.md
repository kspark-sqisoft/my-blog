# ADR-0010: Post 목록 페이지네이션을 offset/page 기반으로 채택

## 상태 (Status)

Accepted - 2026-06-02

## 컨텍스트 (Context)

독자는 발행된 Post 목록을 본다(PRD M3). 목록 페이지네이션 방식을 정해야 한다. 1인 운영 블로그라 Post 볼륨이 낮고 증가 속도가 완만하다. NF3(읽기 성능)은 빠른 목록 로딩을 요구하지만 대규모 데이터셋은 아니다.

평가 기준: 구현 단순성, 데이터 볼륨, UI 요구(페이지 번호).

## 결정 (Decision)

`page`/`pageSize` 기반 offset 페이지네이션을 채택한다(`GET /api/posts?page=&pageSize=`). 응답에 `total`을 포함해 페이지 번호 UI를 지원한다.

## 결과 (Consequences)

긍정:
- 구현이 단순하고 페이지 번호·전체 건수 표시에 적합하다.
- 저볼륨 환경에서 성능 문제가 없다.

부정/감수해야 할 것:
- 데이터가 매우 커지면 깊은 offset의 성능 저하와 페이지 경계 중복/누락 가능성이 있다.

## 검토 시점

Post 수가 크게 늘거나 무한 스크롤·실시간 피드 요구가 생기면 cursor 기반 페이지네이션으로 재평가한다.

# ADR-0013: Comment 답글을 2단계(깊이 2)까지 허용

## 상태 (Status)

Accepted - 2026-06-02

## 컨텍스트 (Context)

기존에는 답글을 1단계(깊이 1)로만 허용했다(ADR-0007, glossary "1단계 한정", PRD 범위 외). 운영 중 더 자연스러운 대화 흐름을 위해 답글의 답글까지 허용하기로 한다. 즉 `Comment → 답글 → 답글의 답글`(깊이 2)을 허용하고 그 이상은 금지한다. 이는 ADR-0007을 대체하는 결정이며, glossary의 Comment 정의도 함께 갱신해야 한다.

평가 기준: 대화 표현력, 깊이 제약의 신뢰성, UI·정렬 복잡도.

## 결정 (Decision)

Comment 답글을 **깊이 2까지** 허용한다. DB는 기존 `parentId` 자기참조를 유지하고, 깊이 제약은 애플리케이션(서비스 계층)에서 강제한다.

- 답글 작성 시 부모 Comment의 깊이를 계산해 **결과 깊이가 2를 초과하면 `400`** 으로 거부한다.
- 깊이 정의: 최상위 Comment = 깊이 0, 그 답글 = 깊이 1, 답글의 답글 = 깊이 2.
- 조회 응답은 깊이 2까지의 중첩 구조로 직렬화한다.

이 결정은 **ADR-0007을 supersede** 한다.

## 결과 (Consequences)

긍정:
- 답글의 답글까지 가능해 대화 맥락이 더 잘 드러난다.
- 스키마(자기참조)는 그대로 유지되어 마이그레이션 부담이 작다.

부정/감수해야 할 것:
- 깊이 계산·검증 로직과 중첩 직렬화가 1단계보다 복잡해진다.
- 프론트의 중첩 렌더링·들여쓰기 UI가 한 단계 더 깊어진다.
- glossary·PRD·TRD의 "1단계" 기술을 모두 "2단계"로 정합화해야 한다.

## 검토 시점

더 깊은 스레드(무한 중첩)나 평면화 표시 요구가 생기면 깊이 정책과 저장 모델(예: path/materialized tree)을 재평가한다.

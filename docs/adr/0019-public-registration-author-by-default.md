# ADR-0019: 공개 회원가입 기본 역할을 AUTHOR로 (가입 즉시 글 작성)

## 상태 (Status)

Accepted - 2026-06-04

이 ADR은 **ADR-0018의 일부 결정을 갱신(amend)** 한다:
- §1의 "`MEMBER` — 글 작성 불가" 전제 중 *공개 가입자의 기본값* 부분,
- §2의 "공개 회원가입 → 기본 역할 **MEMBER**",
- §4의 "운영자 전용 Post 목록은 **ADMIN 한정**"
세 항목을 본 ADR로 대체한다. 역할 enum(ADMIN/AUTHOR/MEMBER) 자체와 JWT 신선도(§3), 소유권 검사 위치(§4 소유권 부분), 마이그레이션(§5)은 그대로 유지된다.

## 컨텍스트 (Context)

ADR-0018은 스팸/봇 가입 표면을 줄이려고 공개 가입자를 글을 못 쓰는 `MEMBER`로 만들고, 운영자(ADMIN)가 수동으로 `AUTHOR`로 승격해야 글을 쓸 수 있게 했다. 그러나 본 블로그의 운영 모델은 "**가입한 사람은 누구나 자기 글을 쓰는 다중 작성자 블로그**"에 가깝다. 매 가입자를 운영자가 일일이 승격하는 흐름은 마찰이 크고, 1인 운영자 병목을 만든다.

결정해야 할 것: 공개 가입의 기본 역할, 작성자가 자기 글을 관리(목록/수정/발행/삭제)하는 경로, 그리고 가입자 쓰기 허용으로 사라지는 스팸 방어선의 대체.

## 결정 (Decision)

### 1. 공개 회원가입 기본 역할 = `AUTHOR`
`POST /api/auth/register`로 생성되는 사용자의 기본 역할을 **`AUTHOR`** 로 한다(기존 `MEMBER`). 가입 즉시 본인 글을 작성/수정/발행/삭제할 수 있다. 클라이언트가 role을 지정할 수 없는 제약(화이트리스트, ADR-0018 §2)은 유지된다.

- 적용 지점은 **애플리케이션 코드(`AuthService.register`)** 한 곳으로 한정한다.
- `User.role`의 **DB 컬럼 기본값은 `MEMBER`로 유지**한다(방어적 기본값). 계정 생성 경로(register=AUTHOR, seed=ADMIN)는 항상 role을 명시하므로 컬럼 기본값에 의존하지 않으며, 혹시 role을 누락한 미래 경로가 생기더라도 "권한 없는 독자"로 안전하게 떨어지게 한다. → 마이그레이션 불필요.

### 2. `MEMBER` 역할의 의미 재정의 — "강등된 독자"
`MEMBER`는 더 이상 *가입 직후 상태*가 아니라, **운영자가 글쓰기 권한을 회수(강등)한 사용자**의 역할이 된다(로그인 독자, 실명 댓글만 가능). 역할 enum은 3단계 그대로 유지한다.

### 3. 운영자 Post 목록/상세를 작성자 스코프로
`AdminPostController`(`/api/admin/posts`, `/:id`)를 **`@Roles('AUTHOR', 'ADMIN')`** 로 연다. 서비스가 actor로 결과를 스코프한다:
- `ADMIN` → 전체 글(기존과 동일, 다른 작성자 초안 포함).
- `AUTHOR` → **본인 글만**(`authorId === actor.id`). 단건 상세는 본인 글이 아니면 403(존재하지 않으면 404 먼저 — ADR-0018 §4의 404→403 순서 유지).

이로써 가입자는 자기 대시보드에서 작성→수정→발행→삭제까지 자급자족한다. 소유권 판정은 ADR-0018대로 도메인(서비스)에 둔다.

### 4. 스팸/봇 대응 — 기존 throttler 유지
가입 즉시 쓰기 허용으로 ADR-0018의 "MEMBER(쓰기 불가)" 1차 방어선은 사라진다. 대체로 **이미 적용된 throttler(rate limiter)** 를 1차 방어로 유지한다. 이메일 인증·가입 승인 큐·모더레이션은 이번에도 **범위 외**로 두고, 악용이 실제 문제로 드러나면 별도 ADR로 도입한다.

## 범위 외 (Out of Scope)
- 이메일 인증/가입 승인 워크플로우, 콘텐츠 모더레이션/신고.
- 작성자 간 협업(공동 저자), 세분화 역할(EDITOR 등).
- 대시보드 문구의 역할별 정교화("운영자 대시보드" 헤딩) — 기능에 영향 없어 후속 과제로 남긴다.

## 결과 (Consequences)

긍정:
- 가입 즉시 글쓰기 → 다중 작성자 블로그의 마찰/병목 제거.
- 작성자가 본인 대시보드에서 글을 끝까지 관리(스코프된 목록/상세).
- 변경이 코드 한 지점(register) + 컨트롤러 역할/서비스 스코프로 국소화. 스키마 마이그레이션 불필요.

부정/비용:
- 공개 가입자가 곧 작성자이므로 스팸/저품질 글 표면 증가 → throttler에만 의존(이메일 인증 부재). 악용 시 모더레이션 도입 필요.
- `MEMBER`의 의미가 "가입 기본값"에서 "강등 상태"로 바뀌어, ADR-0018 시기 문서/테스트의 전제를 갱신해야 한다(glossary, auth/e2e 테스트).
- 다른 작성자 글이 서로 보이지 않으므로(스코프), 전체 글 조망은 ADMIN만 가능.

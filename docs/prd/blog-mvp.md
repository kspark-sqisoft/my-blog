# PRD - blog-mvp

> 본 문서는 제품 요구사항만 다룬다. 기술 결정(라이브러리, DB 스키마, API 설계)은 TRD에서 다룬다.
> 용어는 `docs/glossary.md`의 정의를 따른다 (Post, Comment, Tag, Author, User).

## 개요

### Problem Statement
1인 운영자(User이자 Author)가 자신의 글을 부담 없이 발행하고 관리하는 데 어려움이 있다.
작성 시작부터 발행까지의 마찰이 크면 글을 꾸준히 올리기 어렵다.
동시에 로그인 없는 다수 독자가 발행된 Post를 쉽게 찾아 읽고 Comment로 반응할 수 있어야 한다.

### Goal
1인 운영자가 빠르고 간단하게 Post를 작성·발행·관리할 수 있고,
독자는 로그인 없이 Post를 읽고 Tag로 탐색하며 Comment로 참여할 수 있는 최소 블로그를 제공한다.

## 사용자 (페르소나)

- **운영자**: 시스템에 인증된 단일 User. Post와 Comment를 작성하는 Author 역할을 한다. 글 발행·관리의 주체.
- **독자**: 로그인하지 않은 익명 사용자. 발행된 Post를 읽고 Tag로 탐색하며 Comment를 작성한다.

## 사용자 스토리

- 운영자로서, 나는 Post를 작성·수정·삭제하고 싶다. 그래야 내 글을 자유롭게 관리할 수 있다.
- 운영자로서, 나는 작성 중인 Post를 초안으로 두었다가 원할 때 발행하고 싶다. 그래야 완성된 글만 독자에게 노출된다.
- 운영자로서, 나는 Post에 Tag를 붙이고 싶다. 그래야 글을 주제별로 분류할 수 있다.
- 독자로서, 나는 발행된 Post 목록을 보고 상세 글을 읽고 싶다. 그래야 운영자의 글을 소비할 수 있다.
- 독자로서, 나는 Tag로 Post를 걸러 보고 싶다. 그래야 관심 주제의 글만 찾을 수 있다.
- 독자로서, 나는 Post에 Comment를 달고 답글의 답글까지(깊이 2) 대화하고 싶다. 그래야 운영자·다른 독자와 맥락 있게 소통할 수 있다.
- 운영자로서, 나는 이미지를 업로드해 Post 본문에 넣고 싶다. 그래야 글에 시각 자료를 곁들일 수 있다.

## 기능 요구사항

### MUST (MVP 필수)
- **M1. Post 작성·수정·삭제**: 운영자가 Post를 생성·편집·삭제한다.
- **M2. Post 발행 상태**: Post는 초안과 발행 상태를 구분한다. 발행된 Post만 독자에게 노출된다.
- **M3. Post 목록 읽기**: 독자가 발행된 Post 목록을 본다 (최신순). 로그인 불필요.
- **M4. Post 상세 읽기**: 독자가 발행된 Post 한 건의 본문을 읽는다. 로그인 불필요.
- **M5. Tag 분류**: 운영자가 Post에 Tag를 0~5개 붙인다 (glossary: Post당 0~5개).
- **M6. Tag 탐색**: 독자가 특정 Tag가 붙은 발행 Post 목록을 본다.
- **M7. Comment 작성**: 독자가 발행된 Post에 Comment를 단다. 답글의 답글까지(깊이 2) 달 수 있고, 그 이상 중첩은 거부한다 (glossary / ADR-0013).
- **M8. Comment 표시**: Post 상세에서 해당 Post의 Comment 목록과 답글을 깊이 2까지 중첩 표시한다.
- **M9. 운영자 인증**: 운영자만 쓰기 작업(M1·M2·M5·M10)을 수행할 수 있도록 인증한다. 독자용 읽기/Comment 작성은 인증 불필요. [세부 방식 TBD]
- **M10. 이미지 업로드**: 운영자가 이미지를 업로드하고 Post 본문 마크다운에 임베드한다. 저장은 로컬, 추후 S3로 확장 가능한 구조 (ADR-0012). 동영상 등 기타 미디어는 제외.

### SHOULD (있으면 좋음)
- **S1. Comment 본문 길이 제한**: 스팸·남용 방지를 위한 글자 수 상한. [상한값 TBD]
- **S2. 작성자 표시**: Post·Comment에 Author(운영자) 또는 익명 독자 구분 표시.
- **S3. Post 작성 시각·발행 시각 표시**: 독자에게 발행일 노출.

### MAY (추후 고려 가능)
- **A1. Post 초안 미리보기**: 운영자가 발행 전 렌더링 결과를 미리 본다.
- **A2. Comment 작성자 표시 이름 입력**: 익명 독자가 표시 이름을 직접 입력.

## 비기능 요구사항

- **NF1. 보안 - Comment 입력 안전성**: 익명 Comment 입력에 대한 XSS 방지 및 스팸 억제. 운영자 인증 정보 보호.
- **NF2. 보안 - 권한 분리**: 쓰기 작업(M1·M2·M5·M10)은 인증된 운영자만 수행. 미인증 요청은 거부.
- **NF6. 보안 - 업로드 안전성**: 이미지 업로드는 운영자 전용. 파일 타입(이미지 MIME) 화이트리스트와 크기 상한으로 검증. 본문 렌더 시 안전한 `img`만 허용.
- **NF3. 성능 - 읽기 속도**: Post 목록·상세 페이지의 빠른 초기 로딩. [목표 수치 TBD — 예: 첫 화면 1초 내]
- **NF4. 접근성**: 키보드 탐색 가능, 시맨틱 마크업 등 기본 웹 접근성(a11y) 준수.
- **NF5. 반응형/모바일**: 모바일·데스크톱 모두에서 읽기 편한 반응형 레이아웃.

## 범위 외 (이번에 안 할 것)

- 다수 User 회원가입·로그인 (운영자 1인 인증만; 독자는 익명).
- 좋아요/리액션 등 Post·Comment 반응 기능.
- 전문(키워드) 검색 엔진 (Tag 필터만 제공).
- 동영상·오디오 등 이미지 외 미디어 업로드 (이미지는 M10으로 포함).
- Comment 깊이 3 이상 중첩 답글 (깊이 2까지만 허용 — ADR-0013).
- 업로드 이미지의 미사용 파일 정리(가비지 컬렉션) — ADR-0012상 별도 과제.

## 성공 지표

- **발행 소요시간**: 운영자가 새 Post 작성 시작부터 발행까지 걸리는 시간. [목표값 TBD]
- **발행된 글 수**: 일정 기간 동안 발행된 Post 건수 (꾸준한 운영 여부).
- **독자 재방문/체류**: Post 상세 페이지뷰, 체류시간 등 재방문 관련 지표.
- **댓글 참여**: Post당 Comment 수 등 독자 참여 지표.

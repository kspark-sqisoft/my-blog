# PRD - apple-redesign (애플 스타일 리디자인 + 글 목록 대표 이미지)

> 입력 용어: `docs/glossary.md`. 기술 결정은 여기 적지 않는다(→ `docs/trd/apple-redesign.md`).
> 비고: 이 PRD는 이미 구현된 작업(2026-06-04)을 사후에 정식화한 것이다. blog-mvp(30/30) 이후의 첫 후속 기능.

## 개요

- **Problem Statement**: blog-mvp는 기능은 완성됐지만 기본 보라색 시스템 폰트 UI라 브랜드감·가독성이 약하다. 또한 글 목록이 제목·요약 텍스트만 노출해 시각적 매력이 떨어지고, 요약에 마크다운 원문이 그대로 새어 나온다.
- **Goal**: claude.ai/design에서 만든 **애플 스타일 시안("디버그 노트")** 을 실제 프론트엔드에 적용해 일관된 시각 언어(타이포·색·라이트/다크)를 갖추고, 글 목록 카드에 **대표 이미지**를 노출해 매거진형 탐색 경험을 제공한다.

## 사용자 스토리

- As a 독자, I want 글 목록에서 각 글의 대표 이미지와 깔끔한 요약을 보고 싶다, So that 어떤 글인지 빠르게 가늠하고 읽을 글을 고를 수 있다.
- As a 독자, I want 라이트/다크 테마를 직접 전환하고 싶다, So that 환경·취향에 맞게 편히 읽을 수 있다.
- As a 운영자, I want 작성·대시보드·로그인 화면이 일관된 시스템 UI로 보이길 원한다, So that 콘텐츠 관리가 명료하고 신뢰감 있게 느껴진다.

## 기능 요구사항

### MUST
- 공개 화면(홈/상세/태그/로그인)과 운영자 화면(대시보드/에디터)에 애플 스타일 시각 시스템을 적용한다.
- 라이트/다크 **테마** 토글을 제공하고 선택을 브라우저에 보존한다.
- 글 목록 카드에 **대표 이미지**(본문 첫 이미지)를 표시한다. 본문에 이미지가 없으면 플레이스홀더로 대체한다.
- 글 목록 **요약**은 마크다운 표기(이미지/코드/헤딩/링크/강조)가 제거된 평문으로 보여준다.
- 기존 기능·데이터 계약·접근성(역할/라벨)과 동작을 보존한다(회귀 없음).

### SHOULD
- 한글 가독성을 위한 본문 폰트(Pretendard)와 어절 단위 줄바꿈을 적용한다.
- 상세/목록 등 화면 전환 시 공통 네비/푸터 프레임을 유지한다.

### MAY
- 대표 이미지를 운영자가 수동 지정하는 UI (이번 범위 아님, 오픈 이슈).

## 비기능 요구사항
- **성능**: 목록 응답에 대표 이미지·요약 파생을 추가하되 추가 DB 조회가 없어야 한다.
- **접근성**: 기존 역할(role)·라벨(aria-label)·헤딩 구조를 깨지 않는다. 테마 토글은 키보드 포커스 가능.
- **호환성**: 기존 단위/E2E 계약(목록 `li`, `a[href^="/posts/"]`, `.prose img`, 운영자 버튼 라벨 등)을 유지한다.
- **보안**: 대표 이미지 URL은 기존 업로드 서빙 경로(ADR-0012)를 재사용하며 새 신뢰 경계를 만들지 않는다.

## 범위 외
- 대표 이미지 수동 지정/크롭 UI.
- 이미지가 없는 글에 대한 자동 OG 이미지 생성.
- 다국어/접근성 고대비 테마 등 추가 테마.
- 검색·무한스크롤 등 목록 기능 확장.

## 성공 지표
- 목록 카드에서 본문 첫 이미지가 대표 이미지로 노출(이미지 있는 글 기준 100%).
- 요약에 마크다운 기호(`![`, `##`, `` ``` ``)가 노출되지 않음.
- 라이트/다크 토글이 즉시 반영되고 새로고침 후에도 유지됨.
- 기존 단위/E2E 테스트 전부 통과(회귀 0).

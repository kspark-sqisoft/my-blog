# PRD - 리치 텍스트 본문 에디터 (마크다운 → WYSIWYG 전환)

> 본문 작성 UX 를 마크다운 텍스트에서 **리치 텍스트 WYSIWYG** 로 전환한다.
> ADR-0003(본문은 마크다운으로 한정) 을 supersede 한다. 결정 근거는 ADR-0021.

## 배경 (왜)

지금까지 운영자/작성자는 본문을 마크다운 텍스트로 직접 작성했다. 줄바꿈·헤딩·인용 정도는 충분히 빠르지만,
"이 단어만 색을 바꾸자" 또는 "이 부분만 크기를 키우자" 같은 즉시적 시각 강조는 마크다운 표준에 없다.
일반 블로그 작성 흐름을 따라잡고, 작성자가 도구를 학습하지 않아도 결과를 바로 보면서 쓸 수 있게 한다.

## 사용자 (페르소나)

- **작성자(AUTHOR/ADMIN)**: 글을 쓰면서 결과를 바로 보고 싶다. 일부 단어/줄에 시각 강조(색·크기)를 주고 싶다.
- **독자(공개)**: 발행본의 시각 강조가 일관되게 보인다. 외부 위협 없이 안전.

## 기능 요구사항

- **E-R1. WYSIWYG 입력**: 본문 입력이 마크다운 텍스트가 아니라 리치 텍스트 에디터가 된다. 화면에 보이는 그대로 저장·발행된다.
- **E-R2. 표준 서식**: 굵게/기울임/밑줄/취소선/인라인 코드, H1·H2·H3, 글머리표/번호 매기기, 인용, 코드 블록, 링크, 가로 구분선.
- **E-R3. 색/크기**: 선택 영역(단어/문장)에 글자색과 글자 크기를 지정할 수 있다.
  - 색: 8색 팔레트(검정/회색/빨강/주황/노랑/초록/파랑/보라) + "기본" 해제.
  - 크기: 4단(작게/보통/크게/아주 크게) 프리셋.
- **E-R4. 미디어**: 기존 이미지/MP4 비디오 업로드(ADR-0012/0020) 가 그대로 동작한다. 에디터 도구바에서 업로드 → 본문 안에 노드로 삽입.
- **E-R5. 링크**: 선택 텍스트에 URL 지정, 외부 링크는 `target="_blank" rel="noopener noreferrer"` 자동 적용.
- **E-R6. 키보드 단축키**: 일반 단축키(Ctrl/Cmd+B/I/U, Cmd+Z 등) 지원.

## 비기능 요구사항

- **E-NF1. 저장 형식**: 본문은 **sanitize 된 HTML** 로 저장한다(컬럼 `contentHtml`). 클라이언트도 같은 화이트리스트로 한 번 더 sanitize 한 뒤 렌더한다(이중 방어).
- **E-NF2. 보안**: 서버 sanitize 가 마지막 게이트. 화이트리스트 외 태그/속성/이벤트 핸들러/위험 URL 스킴(`javascript:` 등) 은 모두 제거. style 속성은 `color` / `font-size` 만, 값은 화이트리스트로 정규화.
- **E-NF3. 기존 글 보존**: 기존 글(마크다운 본문) 은 일괄 마이그레이션으로 HTML 로 변환되고, 변환 후에도 시각·내용 동일성이 유지된다(헤딩·리스트·이미지·비디오 임베드 포함).
- **E-NF4. 목록 파생값 정합**: 글 목록의 요약(`summary`)·대표 미디어(`coverImageUrl`) 가 HTML 본문에서도 동일하게 파생된다(요약은 평문 200자, 첫 미디어 URL).
- **E-NF5. 접근성**: 키보드 전용 사용자가 도구바를 사용할 수 있다(버튼 tabindex, aria-label).

## 범위 외 (이번에 안 할 것)

- 표(table) 입력 — TipTap Table 확장 별도 후속.
- 협업 편집(yjs collaboration).
- 글 이력/버전 비교.
- 임의 색(hex 입력)/임의 크기(px 입력) — 8색 + 4단 프리셋만 허용.
- 인라인 HTML 직접 입력(작성자가 raw HTML 을 쓰는 모드).
- 마크다운 toggle("리치/마크다운 둘 다 쓰기") UI — 마크다운 전용 페이지를 따로 두지 않는다.
- 이미지 자르기/리사이즈, 비디오 트리밍.

## 인수 기준 (요약)

- 에디터 진입 시 마크다운이 보이지 않고, 시각 그대로의 본문이 보인다. 도구바에 표준 서식 + 색/크기/링크/미디어 업로드 버튼이 있다.
- 한 단어를 드래그로 선택해 빨강 + "크게" 를 지정 → 즉시 반영. 저장 후 상세 화면에서도 동일하게 보인다.
- 기존 발행 글 모두가 마이그레이션 후 헤딩/리스트/이미지/비디오를 그대로 보여준다. 첫 미디어가 비디오인 글은 카드 커버에 첫 프레임이 표시된다(T-WEB-203 호환).
- `<script>` / `onerror=` / `javascript:` URL / unknown style 속성 등은 저장 또는 렌더 시 모두 제거된다.
- 작성 화면에서 마크다운 textarea 는 사라지고, 본문 임베드는 모두 에디터의 도구바 또는 단축키로 한다.

## 오픈 이슈

- 표(table) 도입은 후속 ADR 로.
- 임의 색/크기 입력 요구가 올라오면 화이트리스트 정책을 재평가.
- 글 이력/버전 — 별도 ADR.

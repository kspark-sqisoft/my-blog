# PRD - 다중 사용자 회원가입 + 역할 기반 권한(RBAC)

> blog-mvp의 "운영자 1인" 전제를 확장한다. 원 PRD(`docs/prd/blog-mvp.md`)에서 범위 외였던 "다수 User 회원가입·로그인"을 도입한다. 결정 근거는 ADR-0018.

## 배경 (왜)

현재는 운영자 1인만 인증하고(시드 생성), 독자는 익명이다. 블로그를 여러 사람이 함께 운영(여러 작성자)하고, 독자가 계정으로 참여(실명 댓글)할 수 있게 한다. 운영자는 슈퍼관리자로서 사용자의 역할을 관리한다.

## 사용자 (페르소나)

- **운영자(ADMIN)**: 슈퍼관리자. 전체 Post 관리 + 사용자/역할 관리. 최초 1인은 시드.
- **작성자(AUTHOR)**: 본인 Post를 작성·수정·삭제·발행. 타인 글은 불가.
- **회원(MEMBER)**: 로그인 독자. 글은 못 쓰고, 로그인 상태로 실명 댓글을 단다.
- **익명 독자**: 비로그인. 발행 Post 읽기 + 익명 댓글(기존 그대로).

## 기능 요구사항

- **M-R1. 회원가입**: 누구나 email·password·이름으로 가입 → 역할 `MEMBER`. 중복 email은 거부(409). 가입 즉시 로그인 상태가 된다.
- **M-R2. 로그인/세션**: 기존 로그인(httpOnly 쿠키) 유지. 세션 응답(`/me`)에 `name`·`role` 포함.
- **M-R3. 역할 관리(운영자)**: 운영자가 사용자 목록을 보고 역할을 변경(MEMBER↔AUTHOR↔ADMIN). **마지막 ADMIN은 강등/삭제 불가**.
- **M-R4. 글 소유권**: AUTHOR는 본인 글만 수정/삭제/발행. ADMIN은 전체. 위반 시 403.
- **M-R5. 글 작성 권한**: 글 생성·업로드는 AUTHOR·ADMIN만. MEMBER/익명은 403/401.
- **M-R6. 회원 실명 댓글**: 로그인 상태로 댓글 작성 시 계정 이름(실명)으로 표시되고 `userId`가 연결된다. 비로그인은 기존 익명(displayName) 병행.

## 비기능 요구사항

- **NF-R1. 권한 즉시성**: 역할 승격/강등은 다음 요청부터 즉시 반영(JWT 만료 대기 없음 — validate에서 DB 재조회).
- **NF-R2. 보안**: 가입 비밀번호 bcrypt 해시(평문 미저장), 클라이언트가 role 임의 지정 불가, 토큰에 역할 비노출, 삭제 계정 토큰 무효(401).
- **NF-R3. 스팸 완화**: 기본 역할 MEMBER(쓰기 불가). 댓글 throttler 유지.
- **NF-R4. 하위호환**: 기존 운영자 계정은 `ADMIN`으로 백필되어 기존 동작 그대로. 기존 익명 댓글 UX 유지.

## 범위 외 (이번에 안 할 것)

- 이메일 인증(가입 메일 확인)·비밀번호 재설정 (SMTP 인프라 부재 — 별도 ADR로 추후).
- 세분화 역할(EDITOR/CONTRIBUTOR), permission 테이블, 소셜 로그인.
- AUTHOR 전용 "내 글" 대시보드(현재는 ADMIN 전용 대시보드 + 편집 화면 직접 진입).
- 회원 프로필 편집 화면, 회원 탈퇴 UI.

## 인수 기준 (요약)

- 비회원이 `/register`로 가입 → MEMBER로 로그인됨. 중복 email 409.
- 운영자가 `/admin/users`에서 MEMBER를 AUTHOR로 승격 → 해당 사용자가 즉시 글 작성 가능.
- AUTHOR가 타인 글 수정 시도 → 403. 본인 글은 가능. ADMIN은 전체 가능.
- 로그인 회원이 댓글 작성 → 계정 이름으로 표시 + userId 연결. 비로그인은 익명 유지.
- 마지막 ADMIN 강등 시도 → 409로 차단.

## 오픈 이슈

- 가입 시 이메일 중복 안내(409 "이미 사용 중") vs 사용자 열거 방지(일반 메시지) — 1차는 UX 우선 409 허용.
- 회원 로그인 시 댓글 throttle 완화 여부 — 후속.

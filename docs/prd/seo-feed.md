# PRD — seo-feed (피드 · 사이트맵 · Open Graph)

> Phase 2 산출물. 용어는 `docs/glossary.md`(피드·사이트맵·Open Graph·대표 이미지·슬러그 등)를 따른다.
> 기술 결정(라이브러리·라우트 서빙 위치·SSR 방식)은 이 문서에 넣지 않는다 — TRD(`docs/trd/seo-feed.md`)와 ADR-0026의 일이다.

## 개요

### Problem Statement
블로그에는 충분한 발행 콘텐츠(슬러그 URL·대표 이미지·평문 요약·작성자까지 갖춤)가 있지만, 그 콘텐츠가 **발견·구독·공유되는 경로가 없다**.

1. **발견(검색)**: 검색엔진에 색인 대상 URL 목록을 알려줄 사이트맵이 없어 색인이 느리고 누락된다.
2. **공유(SNS)**: 글 링크를 카톡·슬랙·트위터에 붙여도 제목·요약·썸네일 카드가 뜨지 않는다. 현재 web 은 SPA(CSR)라 크롤러가 읽는 초기 HTML 에 메타가 비어 있다.
3. **구독(피드)**: 정기 독자가 새 글을 받아볼 표준 피드(RSS)가 없다.

### Goal
발행된 Post 를 **검색엔진·SNS·피드리더가 올바르게 읽도록** 표준 산출물(사이트맵·Open Graph 메타·RSS 피드)을 제공한다. 새 도메인 모델 없이 기존 발행 데이터(슬러그·요약·대표 이미지·작성자·발행일)를 재사용한다.

## 사용자 스토리

- **As a** 검색엔진 크롤러, **I want** 사이트의 색인 대상 URL 과 마지막 수정 시각 목록을 한 곳에서 받기를, **So that** 새 글·변경 글을 빠르고 누락 없이 색인한다.
- **As a** 글을 공유하는 독자, **I want** 글 링크를 SNS 에 붙이면 제목·요약·대표 이미지 카드가 보이기를, **So that** 받는 사람이 클릭 전에 내용을 알 수 있다.
- **As a** 정기 독자, **I want** 블로그를 RSS 피드리더로 구독하기를, **So that** 사이트를 매번 방문하지 않아도 새 글 알림을 받는다.
- **As a** 블로그 운영자, **I want** 표준을 준수하는 산출물을, **So that** 별도 SEO 도구 없이도 검색·공유 노출이 개선된다.

## 기능 요구사항

### MUST
- **M1 (사이트맵)**: `sitemap.xml` 을 제공한다. **발행된 Post 의 슬러그 URL + 태그 페이지(`/tags/:name`) + 홈 등 주요 정적 페이지**를 포함하고, 각 항목에 `lastmod` 를 넣는다(글은 `Post.updatedAt`, 없으면 `publishedAt`; 태그·정적 페이지의 `lastmod` 는 선택). 초안·미발행 글은 절대 포함하지 않는다. sitemaps.org 0.9 스키마를 따른다.
- **M2 (robots)**: `robots.txt` 를 제공하고 그 안에 `sitemap.xml` 의 절대 URL 을 명시한다.
- **M3 (공유 카드 — 서버측)**: 글 상세 링크를 SNS 에 공유하면 **크롤러가 자바스크립트 실행 없이도** 제목·요약·대표 이미지·canonical URL 을 읽을 수 있어야 한다. 즉 서버측에서 발행글 상세의 Open Graph/Twitter 메타를 제공한다(구현 방식은 오픈 이슈/TRD). 미발행·없는 글은 공유 메타를 노출하지 않는다.
- **M4 (OG 내용)**: 공유 메타는 `og:title`(글 제목), `og:description`(평문 요약), `og:image`(대표 이미지 절대 URL, 없으면 사이트 기본 이미지), `og:url`(슬러그 canonical 절대 URL), `og:type=article`, `twitter:card=summary_large_image` 를 포함한다.
- **M5 (피드)**: `feed.xml`(RSS 2.0)을 제공한다. **최근 발행글**을 발행일 최신순으로 담고, 각 항목에 제목·**평문 요약**(본문 전문 아님)·링크(슬러그 절대 URL)·발행일(`pubDate`)·작성자·`guid`(슬러그 절대 URL 재사용, 항목 고유 식별 — 피드리더 중복 방지)를 넣는다.
- **M6 (피드 자동발견)**: 페이지 `<head>` 에 `<link rel="alternate" type="application/rss+xml">` 을 두어 피드리더·브라우저가 피드를 자동 발견하게 한다(적어도 홈·목록에서 발견 가능해야 한다).
- **M7 (XML 안전성)**: 제목·요약·작성자 등 사용자 콘텐츠는 XML/HTML 이스케이프하여 피드·사이트맵·메타가 깨지거나 주입에 악용되지 않게 한다.

### SHOULD
- **S1**: 피드·사이트맵 응답을 캐시(또는 약한 캐시 헤더)하여 매 요청마다 전체를 재생성하지 않는다.
- **S2**: 글 상세 외 목록/홈에도 사이트 기본 Open Graph 메타(사이트명·설명·기본 이미지)를 제공한다.
- **S3**: 피드 항목 수에 상한을 둔다(예: 최근 N개). 정확한 N 은 오픈 이슈.
- **S4**: `og:image` 가 없을 때 사용할 사이트 기본 공유 이미지를 둔다.

### MAY
- **A1**: JSON-LD(`Article`/`BlogPosting`) 구조화 데이터 추가(추후).
- **A2**: Atom 1.0 피드 병행 제공.
- **A3**: 사이트맵에 작성자 프로필 URL 포함 — 단 공개 작성자 프로필(author-profile 기능) 출시 이후. 이번 범위에서는 제외.

## 비기능 요구사항

- **표준 준수**: RSS 2.0, sitemaps.org 0.9, Open Graph protocol, Twitter Cards 규격을 따른다(검증기 통과).
- **성능**: 피드/사이트맵 생성이 목록 조회 수준의 부하를 넘지 않는다(S1 캐시 권장). 추가 N+1 쿼리를 만들지 않는다.
- **보안**: 발행글만 노출(초안·미발행 절대 비노출). 사용자 콘텐츠 이스케이프(M7). 절대 URL 생성을 위한 사이트 base URL 은 환경변수로 주입(하드코딩 금지).
- **정합성(절대 규칙 #9 정신)**: 반환되는 `feed.xml`·`sitemap.xml`·`robots.txt` 가 **실제로 올바른 Content-Type 200 으로 서빙**되는지 쓰기-읽기 왕복으로 검증한다. 특히 공유 메타(M3)는 200/Content-Type 만으로 부족하며 **크롤러 User-Agent 로 받은 글 상세 HTML 에 M4 의 `og:*`/`twitter:*` 필드가 실제로 포함**되는지까지 확인한다. 서빙 경로(API 정적/프록시/nginx)가 세 환경에서 정합해야 한다.
- **일관성**: canonical·OG·피드의 링크는 모두 슬러그 기반 URL(ADR-0022)로 통일한다.

## 범위 외 (이번에 안 함)

- 이메일 뉴스레터/구독 발송(별도 기능).
- 동적 OG 이미지 생성(서버 렌더링 이미지) — 대표 이미지/기본 이미지로 대체.
- 다국어 hreflang, 국제화 SEO.
- 검색엔진 색인 자동 제출(Search Console/IndexNow 핑) — 사이트맵 제공까지만.
- 분석/대시보드(유입·클릭 측정 도구 도입).
- Atom 1.0(MAY 로 보류), JSON-LD(MAY 로 보류).
- 작성자 프로필 URL 의 사이트맵 포함(author-profile 출시 후).

## 성공 지표

- **색인 커버리지**: 발행글 전부가 `sitemap.xml` 에 정확히 노출(초안 0건 포함) — 산출물 검증으로 100% 확인 가능.
- **공유 카드 메타**: 크롤러 UA 로 받은 글 상세 HTML 에 M4 의 6개 필드(`og:title`·`og:description`·`og:image`·`og:url`·`og:type`·`twitter:card`)가 존재하고 값이 정확 — 자동 검증 가능. 추가로 대표 SNS(카톡·페이스북·트위터) 디버거 실제 카드 렌더 확인(출시 검증).
- **피드 유효성**: W3C Feed Validator 등에서 `feed.xml` 통과(오류 0).
- **검색 유입**(측정 도구 도입 시): Search Console 색인 페이지 수·노출 추세 — 측정 수단은 [TBD].

## 오픈 이슈

1. **[중요] 서버측 공유 메타 제공 방식**: SPA(CSR) 에서 크롤러가 OG 를 읽게 하는 방법 — (a) 글 상세의 서버 사이드 렌더/프리렌더, (b) 크롤러(User-Agent) 감지 후 메타만 채운 HTML 제공, (c) 엣지/nginx 레벨 메타 주입 중 택1. TRD + ADR-0026 에서 결정. M3 의 구현 난이도·아키텍처 영향을 좌우한다.
2. **산출물 서빙 위치**: `feed.xml`·`sitemap.xml`·`robots.txt` 를 API(NestJS, `/api` prefix 제외 경로)·web 빌드 정적·nginx 중 어디서 서빙할지(dev 프록시·prod nginx 정합 포함). TRD.
3. **절대 URL 생성**: 사이트 base URL 환경변수 이름·주입 경로(api/web/nginx) 확정. TRD.
4. **피드 항목 수 N**: 최근 몇 개를 담을지(예: 20). 기본값 제안 후 확정.
5. **사이트 기본 공유 이미지(S4)**: 기본 OG 이미지 자산 준비 여부·경로.
6. **lastmod 소스 (해소됨)**: `schema.prisma` 에 `Post.updatedAt @updatedAt` 이 이미 존재 → 글 `lastmod` 는 `updatedAt`(누락 시 `publishedAt`) 사용. 태그·정적 페이지 lastmod 는 선택. (TRD 에서 태그 페이지 lastmod = 해당 태그 최신글 기준 여부만 확정)

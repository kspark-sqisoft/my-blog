# PRD - 본문 비디오 업로드 (이미지 업로드 확장)

> blog-mvp 의 이미지 업로드(ADR-0012) 위에 **MP4 비디오 업로드 + 인라인 재생**을 더한다.
> 작성자가 글 본문에 짧은 클립(데모/녹화/설명 영상)을 넣을 수 있어야 한다는 요구에서 출발.
> 결정 근거는 ADR-0020.

## 배경 (왜)

지금까지 본문 첨부는 이미지(`POST /api/uploads` + 마크다운 `![](url)`) 뿐이라 데모 클립이나
화면 녹화를 넣으려면 외부 호스팅(YouTube 임베드 등)에 의존해야 했다. 글 작성 흐름을 끊지 않으면서
짧은 클립까지 같은 자리에 올릴 수 있게 한다.

## 사용자 (페르소나)

- **작성자(AUTHOR/ADMIN)**: 글에 데모 클립을 넣고 싶다. 이미지 업로드와 같은 UX 로 동작하길 바람.
- **독자(공개)**: 글 목록에서 어떤 클립인지 한눈에 알고 싶고, 상세에서 재생할 수 있어야 한다.

## 기능 요구사항

- **V-R1. 업로드**: 운영자/작성자가 `POST /api/uploads` 에 MP4 비디오를 올린다. 응답은 기존
  이미지와 동일한 모양(`{ url, contentType, size }`)을 따른다.
- **V-R2. 본문 임베드**: 응답 URL 을 본문 마크다운에 `![alt](url)` 로 임베드한다. 별도 syntax 없음.
  렌더러가 URL 확장자가 `.mp4` 면 `<video controls>` 로, 그 외엔 기존대로 `<img>` 로 그린다.
- **V-R3. 목록 카드**: 글 목록(`PostSummaryDto.coverImageUrl`)의 값이 `.mp4` 면 카드 커버
  자리에 비디오의 **첫 프레임**이 멈춰 보인다. 카드 자체로는 재생되지 않고 상세에서만 재생한다.
- **V-R4. 상세 재생**: 글 상세 화면에서 비디오는 표준 컨트롤(재생/일시정지/볼륨/풀스크린)이
  있는 플레이어로 재생된다. 첫 로드 시 자동재생하지 않는다.
- **V-R5. 작성 UX**: 글 에디터의 "이미지 업로드" 버튼이 이미지+비디오 모두 받는다(라벨은
  "미디어 업로드" 로 통일). 다른 포맷은 클라이언트가 막고, 서버도 거부한다.

## 비기능 요구사항

- **V-NF1. 포맷·크기 제한**: MP4(`video/mp4`)만 허용. 크기 상한 50MB(`UPLOAD_MAX_BYTES_VIDEO=52428800`).
  초과 시 413, MIME 불일치는 400. 이미지 상한(5MB)은 그대로 유지.
- **V-NF2. 정적 서빙**: 업로드된 비디오 URL 은 200 + `video/mp4` + `Accept-Ranges: bytes`
  (`<video>` 의 시킹/일시정지 요청에 필요)로 응답해야 한다.
- **V-NF3. 의존성 0 추가**: 서버에 ffmpeg 등 외부 바이너리를 추가하지 않는다. 썸네일은
  브라우저의 `<video preload="metadata">` 첫 프레임으로 잡는다.
- **V-NF4. 권한**: 업로드는 `@Roles('AUTHOR','ADMIN')` (ADR-0018 그대로). MEMBER/익명은 401/403.
- **V-NF5. 보안**: 확장자 위조 차단(MIME + 확장자 양쪽 화이트리스트), 마크다운 렌더러는
  외부 도메인 URL 의 `<video>` 도 sanitize 룰에 명시적으로 허용된 src 만 그린다.

## 범위 외 (이번에 안 할 것)

- WebM/MOV 포맷, HLS·DASH 스트리밍, 트랜스코딩.
- 서버 측 썸네일 추출(ffmpeg). 필요해지면 별도 ADR 로 재평가.
- 자동재생(autoplay), 무음 루프 GIF 대체.
- 비디오 자막(track), 캡션, 외부 비디오 임베드(YouTube/Vimeo).
- "어느 비디오가 어느 Post 에 쓰였는지" 추적 — ADR-0012 의 미사용 파일 정리 과제와 동일하게 후속.

## 인수 기준 (요약)

- 작성자가 에디터에서 MP4 50MB 이내 파일을 올리면 마크다운에 `![](url)` 가 삽입되고, 발행
  후 상세 화면에서 컨트롤 달린 플레이어로 재생된다.
- 50MB 초과 MP4 는 413, JPG/PNG 는 기존대로 이미지 응답, ZIP/MOV 등은 400.
- 글 목록 카드에 비디오 첫 프레임이 멈춰 보이고, 카드 안에서는 재생되지 않는다(클릭 시 상세 이동).
- MEMBER 로그인 또는 비로그인의 업로드 시도는 401/403.
- 비디오 URL 을 직접 호출해도 200 + `video/mp4` 로 서빙되고 `Range` 요청에 응답한다.

## 오픈 이슈

- 잠재 후속: 비디오 별 포스터 이미지 지정 syntax (`![poster](posterUrl) ![video](videoUrl)`),
  ffmpeg 기반 서버 썸네일, 트랜스코딩 큐.
- 미사용 파일 정리(가비지 컬렉션) 는 ADR-0012 와 통합해 한 번에 다룬다.

# PRD — comment-moderation (댓글 수정·삭제·모더레이션)

> Phase 2 산출물. 용어는 `docs/glossary.md`(댓글 모더레이션·소프트 삭제·Comment·Author·운영자 등)를 따른다.
> 기술 결정(스키마·삭제 표현 필드·라우트)은 이 문서에 넣지 않는다 — TRD(`docs/trd/comment-moderation.md`)와 ADR-0027의 일이다.

## 개요

### Problem Statement
현재 Comment 는 **작성·조회만** 되고 수정·삭제가 없다.

1. **작성자**: 오타·실수를 고칠 수 없고, 잘못 단 댓글을 지울 수 없다.
2. **운영자**: 스팸·욕설·부적절 댓글을 제거할 수단이 없어 사실상 운영이 마비된다.
3. **트리 안전성**: 답글이 달린 댓글을 그냥 지우면 자식 답글이 고아가 되거나 트리(깊이 2, ADR-0013)가 깨진다.

### Goal
기존 **Actor 소유권 패턴(ADR-0018)** 과 **깊이 2 트리(ADR-0013)** 위에 Comment 의 **수정·삭제·모더레이션**을 더하되, **답글 트리를 보존**한다(조건부 소프트 삭제). 익명 댓글은 식별 불가라 본인 권한을 주지 않는다.

## 사용자 스토리

- **As a** 로그인 댓글 작성자, **I want** 내 댓글을 수정·삭제하기를, **So that** 오타를 고치고 잘못 단 댓글을 거둘 수 있다.
- **As a** 운영자(ADMIN), **I want** 어떤 댓글이든 삭제하기를, **So that** 스팸·부적절 댓글을 모더레이션할 수 있다.
- **As a** 글쓴이, **I want** 내 글에 달린 댓글을 삭제하기를, **So that** 내 글의 대화 공간을 관리할 수 있다.
- **As a** 독자, **I want** 삭제된 댓글 자리에도 답글 맥락이 남기를, **So that** 대화 흐름이 끊기지 않는다.
- **As a** 익명 댓글 작성자, **I want** 내 익명 댓글은 (식별 수단이 없으므로) 운영자·글쓴이만 지울 수 있음을 이해하기를, **So that** 모더레이션은 가능하되 타인 사칭 삭제는 방지된다.

## 기능 요구사항

### MUST
- **M1 (수정 — 본인만)**: 로그인 작성자 **본인만** 자기 Comment 의 본문(body)을 수정한다. 수정되면 **"수정됨"** 으로 표시한다. 시간 제한은 없다. 운영자·글쓴이도 **남의 댓글 내용은 고치지 못한다**(삭제만 가능). "수정됨" 판정을 위해 **수정 추적 필드 도입이 불가피**하다(현 Comment 에 수정 시각 필드 없음 — 방식은 TRD/ADR-0027).
- **M2 (삭제 권한)**: Comment 삭제는 **본인(로그인 `userId` 일치) + 운영자(ADMIN) + 글쓴이(대상 Post 의 Author)** 만 가능하다.
- **M3 (조건부 삭제)**: 삭제 시 **직계 답글(자식)이 없으면 완전 삭제(하드)**, **직계 답글이 있으면 소프트 삭제**("삭제된 댓글입니다"로 본문·작성자를 가리고 트리·답글은 보존). "답글 있음"의 기준은 **직계 자식 존재**(깊이 2라 서브트리와 실질 동일).
- **M4 (익명 댓글)**: `userId` 없는 익명 댓글은 작성자 식별 수단이 없어 **본인 수정/삭제 불가**. **운영자·글쓴이만 삭제**할 수 있다(조건부 삭제는 동일 적용).
- **M5 (권한·존재 가드)**: 타인 댓글 수정/삭제는 **403**, 없는 댓글은 **404**, 미인증은 **401**. 평가 순서는 **401(미인증) → 404(없음) → 403(권한)** (ADR-0018 규칙 따름).
- **M6 (소프트 삭제 표현)**: 소프트 삭제된 Comment 는 본문·작성자 이름·아바타를 노출하지 않고("삭제된 댓글"), 답글(replies)은 그대로 노출한다.

### SHOULD
- **S1**: 수정은 본문(body)만 바꾼다. `parentId`·깊이·작성자는 불변.
- **S2**: 소프트 삭제된 Comment 에는 **새 답글을 달 수 없다**(이미 삭제된 대화에 추가 방지). 정확한 차단 위치는 오픈 이슈.
- **S3**: 수정·삭제 엔드포인트에도 throttler 를 적용한다(남용 방지 — ADR-0009. 현재 throttler 는 작성 POST 에만 적용되므로 신규 표면).

### MAY
- **A1**: 수정 이력 보관.
- **A2**: 삭제 복구(undelete).
- **A3**: 신고(report) 기반 모더레이션 큐.

## 비기능 요구사항

- **보안·권한**: 소유권·역할은 **서버에서 판정**한다(클라이언트 신뢰 금지). 익명 댓글에는 본인 권한을 부여하지 않는다(식별 불가). 권한 매트릭스는 e2e 로 검증.
- **트리 무결성**: 조건부 삭제로 깊이 2 트리(ADR-0013)를 절대 깨지 않는다. 소프트 삭제 노드도 `replies`·`depth` 를 유지.
- **일관성·BC 정합**: 기존 Actor 소유권 패턴(ADR-0018)을 재사용한다(Post 와 동형). 글쓴이 권한은 Conversation 이 `Comment.postId` 로 대상 Post 의 `authorId` 만 **ID 참조로 조회**해 actor 와 비교한다(Post 객체 직접 보유·역방향 의존 없음 — bounded-contexts 정합). PII: 삭제 시 작성자 표시는 가리되 데이터 보존 정책은 TRD.
- **성능**: 삭제 시 답글 존재 확인은 추가 N+1 없이 단일 조회로.
- **접근성**: "삭제된 댓글"·"수정됨" 표시는 시각만이 아니라 텍스트로 식별 가능.

## 범위 외 (이번에 안 함)

- 신고(report) 기능·모더레이션 큐.
- 댓글 수정 이력·버전 관리.
- 금지어/스팸 자동 필터.
- 익명 댓글 비밀번호(작성 시 비번 → 본인 삭제) — 도입하지 않는다.
- 삭제 복구(undelete).
- 익명→로그인 전환 시 과거 익명 댓글 소유권 연결.

## 성공 지표

- **권한 매트릭스 정확도**: (본인/운영자/글쓴이/타인/익명) × (수정/삭제) 조합이 모두 의도한 200/403/401/404 로 동작 — e2e 로 100% 검증 가능.
- **트리 보존**: 답글 있는 댓글 삭제 시 답글이 모두 살아있고 "삭제된 댓글" placeholder 노출. 답글 없는 댓글 삭제 시 완전 제거.
- **모더레이션 가능성**: 운영자가 임의 댓글(익명 포함)을 삭제 가능.

## 오픈 이슈

1. **삭제 표현 방식**: 소프트 삭제를 어떻게 표현할지 — `deletedAt` 타임스탬프 vs status 필드 vs body 치환. 스키마 영향. TRD/ADR-0027.
2. **"수정됨" 판정**: `updatedAt > createdAt` 비교 vs 별도 `editedAt`/`isEdited` 플래그. 기존 Comment 에 `updatedAt` 이 있는지 확인 필요. TRD.
3. **소프트 삭제 댓글에 답글 차단 위치(S2)**: 작성(create) 시 부모가 소프트 삭제 상태면 거부할지. TRD.
4. **완전 삭제 vs 소프트 경계 (확정)**: "답글 있음" = **직계 자식 존재** 기준(M3). 깊이 2라 서브트리와 실질 동일하므로 TRD 는 직계 자식 카운트로 구현한다.
5. **삭제된 댓글의 카운트/정렬**: 소프트 삭제 노드를 목록 카운트에 포함할지(현재 별도 카운트 없음 — 영향 적음).

# PRD — author-profile (공개 작성자 프로필)

> Phase 2 산출물. 용어는 `docs/glossary.md`(작성자 프로필·소개(bio)·프로필·아바타·대표 이미지 등)를 따른다.
> 기술 결정(스키마·라우트·목록 API 재사용 여부)은 이 문서에 넣지 않는다 — TRD(`docs/trd/author-profile.md`)와 ADR-0028의 일이다.

## 개요

### Problem Statement
다저자 블로그(RBAC AUTHOR 다중 — ADR-0018)인데, 글 목록·상세에 **작성자 이름은 보이지만** 그 작성자를 **클릭해도 그가 쓴 다른 글이나 소개를 볼 경로가 없다**. 독자는 마음에 든 글의 저자를 더 탐색할 수 없고, 작성자는 자기 글 모음·소개를 공개할 페이지가 없다.

### Goal
각 작성자(User)의 **공개 프로필 페이지**(`/users/:id`)를 제공해, 이름·아바타·소개(bio)·가입일·발행글 수와 **그가 쓴 발행글 목록**을 보여준다. 글 목록·상세의 작성자 이름을 이 프로필로 연결한다. 기존 아바타(ADR-0025)·발행 목록 읽기 모델을 재사용하고, 발행글만 노출한다(초안 비노출, 이메일 비노출).

## 사용자 스토리

- **As a** 독자, **I want** 글의 작성자 이름을 눌러 그 작성자의 프로필(소개 + 다른 발행글)을 보기를, **So that** 마음에 든 저자의 글을 더 찾아 읽는다.
- **As a** 작성자, **I want** 내 이름·아바타·소개와 내가 쓴 글 모음이 공개 페이지로 모이기를, **So that** 내 글을 한 곳에서 보여주고 나를 소개한다.
- **As a** 작성자, **I want** 내 소개(bio)를 직접 편집하기를, **So that** 프로필에 나를 표현한다.

## 기능 요구사항

### MUST
- **M1 (공개 프로필 조회)**: `User.id`(cuid)로 작성자 공개 프로필을 조회한다. 응답은 **이름·아바타·소개(bio)·가입일·발행글 수**를 포함하고 **이메일은 절대 포함하지 않는다**. 없는 id 는 404.
- **M2 (작성자 발행글 목록)**: 프로필은 그 작성자의 **발행글 목록**을 페이지네이션으로 보여준다. **초안·미발행은 절대 포함하지 않는다**. 정렬은 발행일 최신순(기존 목록과 동일).
- **M3 (작성자 링크)**: 글 목록 카드와 글 상세 헤더의 **작성자 이름을 클릭하면 그 작성자 프로필(`/users/:id`)로 이동**한다. (링크에 필요한 작성자 식별자 노출은 TRD)
- **M4 (소개 편집)**: 작성자 본인이 "프로필"(`/profile`, ADR-0025)에서 **소개(bio)를 편집**한다(이름·아바타와 함께). 빈 소개는 미표시.
- **M5 (소개 노출)**: 작성자 프로필에 소개(bio)가 있으면 보여주고, 없으면 해당 영역을 숨긴다.

### SHOULD
- **S1**: 프로필에 발행글이 없으면 "아직 발행한 글이 없습니다" 같은 빈 상태를 보여준다.
- **S2**: 작성자 프로필 페이지는 SEO 메타(seo-feed)의 기본 OG 메타 대상이 될 수 있으나, 동적 OG·사이트맵 포함은 **이번 범위 외**(seo-feed 오픈이슈에서 author-profile 출시 후로 미뤘음).
- **S3**: 소개(bio)는 길이 상한을 둔다(스팸·레이아웃 보호). 정확한 값은 TRD.

### MAY
- **A1**: 작성자 프로필에 좋아요 합계·댓글 수 등 활동 통계.
- **A2**: 프로필 커버 이미지.
- **A3**: 예쁜 핸들(username) URL — 이번엔 `:id`(cuid), 핸들은 후속.

## 비기능 요구사항

- **보안·PII**: 공개 프로필·발행글 목록 모두 **이메일 비노출**(이름·아바타·bio 만). 발행글만(초안·미발행 격리). bio 는 표시 시 XSS 안전하게 처리(평문/escape).
- **일관성**: 발행 목록은 기존 발행 목록 읽기 모델(정렬·페이지네이션, ADR-0010)과 동일 규칙. 아바타는 ADR-0025 재사용. 본인 편집은 기존 `PATCH /api/auth/me`(ADR-0025) 확장.
- **성능**: 프로필 조회·발행글 목록이 추가 N+1 을 만들지 않는다. 발행글 수는 count 1회.
- **접근성**: 작성자 이름 링크는 키보드 포커스·스크린리더로 식별 가능.
- **정합성(절대 규칙 #9 정신)**: 프로필이 참조하는 아바타 URL 이 실제 200 으로 서빙되는지(기존 아바타 검증 재사용).

## 범위 외 (이번에 안 함)

- 팔로우/구독, 작성자 알림.
- 작성자 활동 통계 대시보드(좋아요·댓글 합계).
- 프로필 커버 이미지.
- 예쁜 핸들(username) URL — `:id`(cuid)만.
- 댓글 작성자 이름 → 프로필 링크(이번엔 글 작성자만).
- 작성자 프로필의 동적 OG·사이트맵 포함(seo-feed 후속).
- 비공개/비활성 작성자 처리(전 작성자 공개).

## 성공 지표

- **프로필 정확도**: `/users/:id` 가 그 작성자의 발행글만(초안 0건) 정확히 노출, 이메일 0 노출, **발행글 수가 실제 발행 count 와 일치**(count 1회, N+1 없음) — 산출물·테스트로 검증.
- **탐색 동선**: 글 목록·상세에서 작성자 이름 클릭 → 프로필 이동이 동작.
- **소개 왕복**: 본인이 bio 편집 → 프로필에 반영(쓰기-읽기 왕복).

## 오픈 이슈

1. **프로필 조회 API 형태**: 프로필 메타와 발행글 목록을 한 응답(`GET /api/users/:id`)으로 줄지, 메타와 목록을 분리(`/api/users/:id` + 기존 `/api/posts?author=:id`)할지. TRD.
2. **작성자 식별자 노출(M3)**: 글 목록 카드 링크를 위해 `PostSummaryDto` 에 작성자 id 를 노출해야 한다(현재 authorName 만). 노출 필드명·범위는 TRD.
3. **bio 길이 상한(S3)**: 구체 값(예: 200자). TRD/ADR-0028.
4. **bio 마이그레이션**: `User.bio` 추가(nullable, 추가 전용). TRD.
5. **(확정) 전 User 공개**: 발행글 0인 MEMBER 도 프로필 페이지를 갖는다(빈 목록 — S1). 명칭은 "작성자 프로필"로 유지한다(글 유무와 무관한 User 공개 페이지). 비공개/비활성 작성자 처리는 범위 외.
6. **ADR-0025 amend/supersede**: ADR-0028 이 ADR-0025 의 (a) `UpdateProfileDto`(name·avatarUrl) 계약을 **bio 포함으로 amend**, (b) **"범위 외: 공개 프로필 페이지(타인이 보는)" 항목을 supersede**(이번에 구현) 함을 ADR 단계에서 명시한다(Accepted ADR 변경 추적).
7. **작성자 프로필 조립 주체**: Auth 가 Publishing 을 의존하면 "Auth 최하위" 불변식이 깨진다 → 프로필 메타(`/api/users/:id`, Auth)와 작성자 발행글 목록(`/api/posts?author=:id`, Publishing)을 **분리**하고 **web 프레젠테이션에서 조합**한다(BC 정합). TRD 확정.

# PRD — 시리즈 (Series, 연재)

> 용어는 `docs/glossary.md` 정의를 따른다(시리즈·시리즈 순서·시리즈 네비게이션·Post·작성자·발행).

## 개요

### Problem Statement
블로그에 연재물(예: "React 입문 1·2·3편", "DDD 실전 시리즈")이 늘면, 독자는 **다음 편을 찾기 어렵고** 작성자는 **연재 묶음을 보여줄 방법이 없다**. 현재는 글이 발행일 역순으로 흩어져 있어 순서대로 읽는 동선이 없다.

### Goal
작성자가 자기 글을 **순서 있게 시리즈로 묶고**, 독자가 **시리즈 단위로 발견·정주행**할 수 있게 한다. 한 글은 최대 한 시리즈에 속하며, 순서는 작성자가 수동 지정한다.

## 사용자 스토리

- As a **작성자(AUTHOR/ADMIN)**, I want **내 발행글들을 하나의 시리즈로 묶고 순서를 정하고 싶다**, so that **독자가 연재를 순서대로 읽을 수 있다**.
- As a **독자**, I want **글 상세에서 이 글이 속한 시리즈와 이전/다음 편을 보고 싶다**, so that **다음 편으로 막힘없이 이어볼 수 있다**.
- As a **독자**, I want **시리즈 상세 페이지에서 그 연재의 전체 글을 순서대로 보고 싶다**, so that **처음부터 정주행할 수 있다**.
- As a **독자**, I want **시리즈 목록(인덱스)에서 어떤 연재가 있는지 훑어보고 싶다**, so that **관심 연재를 발견할 수 있다**.
- As a **운영자(ADMIN)**, I want **모든 시리즈를 관리(수정·삭제)할 수 있다**, so that **다작성자 블로그의 연재를 운영할 수 있다**.

## 기능 요구사항

### MUST
- **M1. 시리즈 생성/수정/삭제(작성자 소유)**: 작성자가 제목(필수)·설명(선택)으로 시리즈를 만들고 수정·삭제한다. 본인 시리즈만(ADMIN 은 전체). 삭제 시 소속 글의 시리즈 연결만 해제되고 **글 자체는 삭제되지 않는다**.
- **M2. 글의 시리즈 소속·순서 지정**: 작성자가 **자기 글**을 자기 시리즈에 넣고/빼고, 시리즈 안에서 **순서를 수동 지정**한다. 한 글은 **최대 한 시리즈**에만 속한다.
- **M3. 시리즈 상세(공개)**: 시리즈 제목·설명 + 그 시리즈의 **발행글을 순서대로** 보여준다. 초안은 노출하지 않는다. 발행글이 없으면 빈 상태 안내.
- **M4. 글 상세의 시리즈 네비게이션(공개)**: 시리즈에 속한 발행글 상세에서 "X 시리즈 · N/M편"과 **이전/다음 발행글 링크**를 보여준다. 시리즈 미소속 글에는 노출하지 않는다.
- **M5. 시리즈 목록/인덱스(공개)**: 전체 시리즈를 목록으로 보여준다(제목·설명·글 수 등). 페이지네이션(ADR-0010 동형).

### SHOULD
- **S1. 빈/경계 상태 처리**: 발행글 0개 시리즈는 상세에서 "아직 발행된 글이 없습니다" 안내. 시리즈 목록이 비면 안내 문구.
- **S2. 작성자 관리 동선**: 작성자가 글 편집 화면 또는 시리즈 관리 화면에서 소속·순서를 직관적으로 바꾼다(드래그 또는 순서값).
- **S3. 발견성**: 시리즈 상세/목록에서 작성자 프로필(ADR-0028)·태그와 자연스럽게 연결.

### MAY
- **M(may)1. 시리즈 대표 이미지**: 첫 글의 대표 이미지를 시리즈 카드 썸네일로 파생(ADR-0015/0020 동형). [후속 가능]
- **M(may)2. 시리즈 RSS/사이트맵 노출**(seo-feed 확장). [범위 외, 후속]

## 비기능 요구사항
- **성능**: 시리즈 상세·네비·목록은 N+1 없이 인덱스를 탄 쿼리로(시리즈 내 정렬은 `(seriesId, seriesOrder)` 인덱스). 목록은 페이지네이션.
- **보안/권한**: 시리즈 생성·수정·삭제·멤버십 변경은 **소유 작성자 또는 ADMIN**만(Actor 소유권 패턴, ADR-0018). 공개 조회는 **발행글만** 노출(초안 격리). 타인 글을 내 시리즈에 임의로 넣을 수 없다.
- **접근성**: 시리즈 네비·목록 링크는 접근 가능한 이름을 가진다(키보드·스크린리더).
- **일관성**: URL 식별자는 슬러그(ADR-0022 동형). 검증은 각 패키지(ADR-0004) — 폼 zod 는 web 인라인, 서버는 class-validator(함정 #9 — shared 순수성 유지).

## 범위 외 (Out of Scope)
- 한 글이 **여러 시리즈**에 속하기(다대다) — 이번은 최대 1개.
- 시리즈 **구독/알림**(notifications 기능에서 별도 — ADR-0031 예정).
- 시리즈 **RSS/사이트맵/OG** 노출(seo-feed 확장, 후속).
- 시리즈 **접근 제어**(비공개 시리즈) — 모든 시리즈는 공개, 노출 글만 발행 격리.
- 시리즈 내 글의 **자동 정렬**(발행일순 등) — 수동 순서만.

## 성공 지표
- 작성자가 시리즈를 만들고 ≥2개 글을 순서대로 묶을 수 있다(왕복 동작).
- 독자가 시리즈 상세에서 순서대로 글을 열고, 글 상세의 "다음 편"으로 끊김 없이 이동한다.
- 초안/타인 글이 공개 시리즈·네비에 절대 노출되지 않는다(권한·발행 격리 회귀 0).

## 오픈 이슈
- ✅ 해소(→ TRD §3) 시리즈 멤버십·순서 변경 API 형태: **시리즈 전용 `PUT /api/series/:id/posts {postIds}` 원자 재지정**으로 결정(글 수정에 싣지 않음). 소유권 판정·재정렬 트랜잭션을 한 곳에 모음.
- ✅ 해소(→ TRD §결정추출 2) 순서 충돌: `postIds` 배열 **index 가 곧 seriesOrder** 라 중복 order 가 구조적으로 불가(연속 재부여).
- ✅ 해소(→ ADR-0029 §결과/부정) 다수 Post 순서 갱신과 "한 트랜잭션 = 한 Aggregate"의 긴장: 같은 타입(Post) 다건 갱신을 불변식 보존을 위한 **실용적 예외**로 명시 수용.
- ✅ 해소(→ TRD §3·§8, ADR-0029 §6) 멤버십 권한 경계: **AUTHOR 는 본인 소유 글만** 본인 시리즈에 편입, **ADMIN 은 임의 글/시리즈** 편입 가능(운영 모더레이션).

# Handoff — T-WEB-306 (에디터 미디어 미리보기 + 삭제/교체) & T-WEB-305 마감

날짜: 2026-06-05 / feature: rich-text-editor

## 한 일
글 에디터(TipTap RichEditor)에서 이미지/동영상을 "주소 텍스트"가 아니라 **미리보기**로 보여주고,
호버 오버레이로 **삭제/교체**할 수 있게 했다. 레거시(마크다운) 글도 HTML 백필로 정상화.

- **레거시 백필**: 기존 `migrate:md-to-html`(T-INFRA-303) 스크립트를 **dev DB 에 실행**(scanned=31, changed=31, 멱등 재실행 changed=0). contentHtml 이 비어 있던 옛 글의 `![](url)` 마크다운이 `<img>` HTML 로 채워져, 에디터·공개뷰(RichContent) 모두 이미지가 미리보기로 렌더.
- **미디어 NodeView**(`mediaNodeView.ts`): `MediaImage`(=@tiptap/extension-image 확장) + 기존 `Video` 노드에 `addNodeView(mediaNodeView())` 부착. `buildMediaView` 가 미리보기 + 호버 오버레이(🔄 교체 / 🗑 삭제) DOM 생성.
- **클릭 전달 문제 해결**: ProseMirror editable 내부 버튼은 trusted 마우스 이벤트가 자체 click 리스너에 도달하지 않는다(에디터 root 가 가로챔). 그래서 버튼 동작을 DOM 에 실어두고(`__mediaAction` + `data-media-action`), RichEditor 의 `editorProps.handleDOMEvents.mousedown` 이 `runMediaActionFrom` 으로 직접 실행. 삭제=`deleteRange(getPos..)`, 교체=숨김 input → `onUploadMedia` → `setNodeAttribute('src')`. 이미지·동영상 공통.
- **T-WEB-305 동반 해결**: 격리 prod e2e 6 spec 실패의 진짜 원인은 prod 빌드가 아니라 **셀렉터 strict-mode 충돌**(툴바 "제목 1/2/3"↔`getByLabel('제목')`, "본문 서식"↔`getByLabel('본문')`). e2e 를 `{ exact: true }` 로 한정. rich-editor.spec 은 Ctrl+A(전체선택) 직후 이미지 업로드 시 본문 교체되던 시퀀스를 "업로드 전 커서 모으기"로 수정.

## 결정한 것
- 미디어 컨트롤 트리거는 버튼 click 이 아니라 에디터 `handleDOMEvents.mousedown`(ProseMirror 가 확실히 받는 단계). 새 ADR 불필요(ADR-0020/0021 구현 범위).
- 레거시 글은 1회성 백필로 데이터 정규화(클라이언트 변환 대신 — 에디터+공개뷰 일관).

## 알려진 이슈 / 검증 한계
- **헤드리스 Playwright(MCP 상시 세션)는 ProseMirror editable 내부로 마우스 이벤트를 전달하지 못함** → 삭제/교체 "클릭"은 브라우저 자동화로 직접 검증 불가. 대신 (a) 단위 테스트, (b) `__mediaAction()` 직접 호출 시 이미지 제거 확인(before 1 → after 0), (c) operator-flow e2e(이미지 업로드+저장+발행+상세 렌더) 통과로 우회 검증. **실제 브라우저에서 사용자 클릭 동작은 사용자 확인 권장.**

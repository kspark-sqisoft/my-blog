# Handoff — T-INFRA-303 markdown → html 일회성 마이그레이션 스크립트

날짜: 2026-06-05 / feature: rich-text-editor / 근거: ADR-0021

## 한 일
기존 Post 의 `contentMarkdown` 본문을 sanitize 통과 HTML 로 일괄 변환하는 일회성 스크립트와
재사용 가능한 변환 헬퍼를 추가했다. T-INFRA-302 가 추가한 빈 `contentHtml` 컬럼이 이제 모든
기존 글에 대해 채워진다.

- 의존성: `markdown-it`, `sanitize-html` (+ `@types/...`) api 패키지에 추가.
- `packages/api/src/publishing/markdown-to-html.ts`:
  - `convertMarkdownToHtml(md)`: markdown-it(html:true, linkify) → `.mp4` `<img>` → `<video>` 후처리
    → `sanitize-html(richHtmlSchema)`. `<a>` 에 `target=_blank rel=noopener noreferrer` 자동 부착.
  - `sanitizeRichHtml(html)`: 이미 HTML 인 입력의 sanitize 만 (T-PUB-301 가 사용 예정).
- `packages/api/scripts/migrate-md-to-html.ts`: PrismaClient 직접 사용. `--dry-run` 지원.
  같은 입력 → 같은 결과 비교 후 변경 필요한 row 만 update.
- `package.json` 에 `migrate:md-to-html` 스크립트 등록.

## 결정한 것
- `markdown-it` 옵션 `html: true` — raw HTML 도 받아 일관되게 sanitize 게이트로 처리(이중 게이트 단순화).
- 변환 헬퍼는 **순수 함수** 로 export — T-PUB-301 의 PostService 가 이를 재사용해 사용자 입력(HTML) 도 같은 sanitize 정책을 받을 수 있다(`sanitizeRichHtml`).
- 외부 링크는 sanitize 의 `transformTags` 단계에서 `target/rel` 자동 부착 — 본문에 의해 의존하지 않고 정책으로 강제.
- 스크립트는 PrismaClient 직접 사용(seed.ts 와 같은 패턴). NestJS 부팅 없이 가볍게.

## 알려진 이슈
- 컬럼 이름이 Prisma camelCase 기본을 따라 DB 에 `"contentHtml"` quoted identifier — 다른 컬럼들과 일관. 후속에 snake_case 통일 시 별도 마이그레이션.
- prisma client 재생성을 `pnpm prisma generate` 로 명시 실행해야 마이그레이션 후 새 컬럼이 인식. 첫 실행에서 `PrismaClientValidationError: Unknown field 'contentHtml'` 로 한 번 막혔다 — handoff 에 명시.
- 우리 운영 글 2 개 외 다른 글은 없으므로 변환 본문은 위 2건만. 실제 prod 가 있다면 글 수에 비례한 시간 소요(작음).

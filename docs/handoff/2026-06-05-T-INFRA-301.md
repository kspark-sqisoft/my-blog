## 결정한 것
- `contentHtml` 은 **과도기 optional**. 호출처(api PostService, web PostEditor) 가 한 번에 required 를 만족할 수 없어, 시리즈의 다음 단계(T-INFRA-302/T-PUB-301) 에서 점진적으로 required 로 회수한다. acceptance 의 "compile enforce"·"contentMarkdown 응답에서 사라짐" 도 T-PUB-301 에서 완성.
- 시각 서식은 **Tailwind 클래스 화이트리스트만** — 인라인 `style` 미허용. CSS injection 표면 최소화 + sanitize 단순화.
- `transformTagNames` 만 shared 에 두고 실제 transform 함수는 서버(sanitize-html) 측에서 attach(클로저·플랫폼 의존). 공유 가능한 정책만 한 곳에 둔다.

## 알려진 이슈
- `richHtmlSchema` 의 모양이 sanitize-html 의 `IOptions` 와 1:1 매핑인지는 다음 태스크(T-PUB-301) 에서 실제 라이브러리 통합 시 한 번 더 점검.
- dompurify 의 옵션 구조는 sanitize-html 과 다름 — 클라이언트 어댑터 변환은 T-WEB-303 에서 구현.

# Handoff — T-PUB-302 파생값 HTML 입력 대응

날짜: 2026-06-05 / feature: rich-text-editor / 근거: ADR-0021

## 한 일
글 목록 카드의 두 파생값(`summary`, `coverImageUrl`) 생성 함수를 HTML 입력 기준으로 교체했다.
ADR-0021 본문 모델 전환의 마지막 백엔드 단계 — 이후 클라이언트 작업(T-WEB-301~304) 으로 넘어간다.

- 의존성: `cheerio` 추가.
- `packages/api/src/publishing/cover-image.ts`:
  - 정규식 기반 마크다운 파서 → **cheerio.load** 기반. DOM 순회로 `<img>`/`<video>` 중 가장 먼저
    src 가 있는 노드의 src 반환. 함수명은 `extractFirstImageUrl` 호환 유지 — 의미는 첫 미디어 URL 의 슈퍼셋(이미지+비디오).
- `packages/api/src/publishing/markdown-summary.ts`:
  - 정규식 기반 마크다운 제거 → **cheerio.text()** 기반. script/style 방어적 제거 후 공백 정리 + max trim.
- `post.service.ts` `toSummary`:
  - `const body = post.contentHtml || post.contentMarkdown` 폴백. 정상 케이스(T-INFRA-303 마이그레이션 후) 는 contentHtml 사용. 빈 contentHtml 인 row 가 있어도 contentMarkdown 으로 안전.
- spec 갱신:
  - `cover-image.spec.ts` 8 케이스 — `<img>` 단일/다중, `<video>` 인식, `<img>+<video>` 순서, src 누락 건너뛰기, 빈 입력.
  - `markdown-summary.spec.ts` 6 케이스 — 헤딩/단락 평문, 미디어 제외, 링크 텍스트만, max trim, 빈 입력, 공백 정리.

## 결정한 것
- **함수명 호환 유지**(`extractFirstImageUrl`, `toSummaryText`) — 호출처 변경 면적 0. 의미는 슈퍼셋(이미지 → 미디어, 마크다운 → HTML).
- **body 폴백 순서**: contentHtml 우선, 없으면 contentMarkdown. 마이그레이션 직후 데이터에서 발생할 수 있는 빈 contentHtml 케이스를 안전하게 다룸.
- 정규식 마크다운 파서를 완전히 제거 — 마크다운 본문은 더 이상 직접 다루지 않는다(파생값 함수에 한정).

## 알려진 이슈
- cheerio 가 stripped HTML 의 평문을 합치면서 단어 사이 공백이 사라질 수 있다(예: `<p>안녕</p><p>본문</p>` → "안녕본문"). 우리 케이스에서는 공백 collapse 로 처리 가능하지만, 후속에 block-level 노드 사이에 공백을 강제하는 정책을 검토할 수 있다.
- listAdminPosts 등 다른 호출처는 `toSummary` 외 별도 파생값을 안 만든다 — 추가 영향 없음.

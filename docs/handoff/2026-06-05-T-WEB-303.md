# Handoff — T-WEB-303 RichContent 렌더러 + Markdown 제거

날짜: 2026-06-05 / feature: rich-text-editor / 근거: ADR-0021

## 한 일
PostDetail 의 본문 렌더러를 마크다운 기반 `<Markdown>` 에서 HTML 기반 `<RichContent>` 로
교체했다. 본문 HTML 은 서버 sanitize-html 통과본이지만, 클라이언트도 dompurify 로 한 번 더
필터해 이중 방어. ADR-0021 의 마지막 핵심 위젯.

- 의존성: `dompurify` + `@types/dompurify` 추가.
- `components/RichContent.tsx`:
  - dompurify 옵션: `ALLOWED_TAGS` (shared 의 `richHtmlSchema.allowedTags` 그대로) + `ALLOWED_URI_REGEXP`(http/https/mailto/tel/상대경로).
  - 모듈 로드 시 `uponSanitizeAttribute` hook 으로 video 의 `controls/preload/playsinline/muted/loading` 을 강제 보존(ADR-0020 호환).
  - `dangerouslySetInnerHTML` — 입력은 항상 sanitize 결과.
  - 단위 spec 7 케이스(`RichContent.test.tsx`).
- `pages/PostDetail.tsx`:
  - `<Markdown content={post.contentMarkdown} />` → `<RichContent html={post.contentHtml || post.contentMarkdown} />` (과도기 폴백).
- `pages/PostDetail.test.tsx` 의 fixture 에 `contentHtml: '<h1>본문 헤딩</h1><p>내용 단락</p>'` 추가 — 헤딩 단언이 통과되도록.

## TDD 사이클 확인
- RED: 7 케이스 작성 → 1 fail("javascript: href 제거" 의 .toMatch 호출이 null 에서 깨짐 — 단언 형식 정밀화 후 통과).
- 추가 발견: full web test 2 fail — PostDetail spec 의 마크다운 헤딩 단언이 안 통하고(fixture 갱신), RichContent 의 video preload 가 null. 후자는 dompurify element-specific 처리.
- 1차 해결 시도: `ALLOWED_ATTR` + `ADD_ATTR` 로 video 속성 추가 → 실패. dompurify 가 ALLOWED_ATTR 모드에서도 video 속성을 임의 차단.
- 최종 해결: 클라 dompurify 는 **태그 화이트리스트**만 강제 + 모듈-스코프 `uponSanitizeAttribute` hook 으로 video 안전 속성 보존. 속성 정밀 화이트리스트는 서버 sanitize-html 이 책임.

## 결정한 것
- **태그 화이트리스트만 클라이언트**: dompurify 의 강점인 태그 필터 + URI 스킴 차단으로 한정. 속성 정밀 화이트리스트는 dompurify 의 element-specific 처리와 충돌(video preload 등) — 서버 sanitize-html(`richHtmlSchema`) 이 정밀 게이트.
- **video 안전 속성 hook**: 글로벌 hook 등록은 모듈 로드 시 한 번. 다른 dompurify 호출처 없음을 가정. 추가 호출처 생기면 hook 영향 확인.
- **PostDetail 폴백**: `contentHtml || contentMarkdown` — 과도기 데이터 안전망. 클라 dompurify 가 마크다운 plain text 를 받으면 그대로 텍스트 노드로 렌더(헤딩 없이) — UX 약화지만 안전.

## 알려진 이슈
- 시각 스타일(`.ab-rich-content`/`.ab-rich-editor`/`.ab-rich-toolbar` CSS) 는 아직 없음. T-WEB-304 e2e 이후 또는 별도 디자인 패스에서 보강.
- dompurify 글로벌 hook 은 모듈 로드 부수효과. 후속에 `createDOMPurify` 인스턴스 분리 가능.
- 우리 dompurify 호출은 클라이언트만 — node 환경에서는 jsdom 의존(이미 vitest 환경에서 동작).

읽기 경험(B) 두 번째 태스크. 본문 HTML 을 렌더타임에 보강(ADR-0023).

## 한 일
- `packages/web/src/lib/article-enhance.ts` — `enhanceArticleHtml(html): Promise<{html, toc}>` 신설.
  - DOMParser 로 파싱 → h1~h3 에 slug id 부여(중복 `-2/-3` 유일화) + `toc[]`(id·text·level) 추출.
  - 코드블록(`pre code`)을 highlight.js **자동감지**로 하이라이트(`hljs` + `language-*` class). highlight.js 는
    `import('highlight.js/lib/common')` 동적 import(번들 분리), 로드 실패 시 평문 graceful.
  - 앵커 slug 는 web 로컬 구현(api `slugify` 는 패키지 경계상 import 불가, 앵커는 문서 내 유일성이 목적).
- `packages/web/src/lib/sanitize-rich-html.ts` — 구 RichContent 의 dompurify sanitize 게이트를 util 로 추출(공유).
- `packages/web/src/components/ReadingArticle.tsx` — sanitize → enhance 렌더러(구 RichContent 대체).
  - 첫 페인트는 sanitize 본문(`dangerouslySetInnerHTML`), enhance 완료 후 같은 노드 `innerHTML` 을 교체
    (effect-내 setState 회피 = lint `set-state-in-effect`/`refs` 준수). 목차는 `onToc` 로 상위 전달(T-READ-103 용).
- `packages/web/src/pages/PostDetail.tsx` — 본문 렌더를 `RichContent` → `ReadingArticle` 로 교체.
- `packages/web/src/index.css` — `.ab-rich-content` 코드 박스 + highlight.js 토큰 테마(라이트/다크).
- 제거: 미사용이 된 `RichContent.tsx`/`RichContent.test.tsx`. 보안 회귀 테스트는 `sanitize-rich-html.test.ts` 로 이전.

## 결정한 것
- 파생 표현(헤딩 id·목차·하이라이트)은 저장 없이 렌더타임 계산 — ADR-0023.
- enhance 결과를 React state 가 아닌 ref+innerHTML 로 반영(lint 규칙 + 추가 리렌더 회피).

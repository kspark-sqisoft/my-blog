# Handoff — 슬러그 URL (T-PUB-108 + T-WEB-309)

날짜: 2026-06-05 / feature: seo (로드맵 A①) / 근거: ADR-0022

## 한 일
글 상세 URL을 불투명한 cuid(`/posts/cmpz9...`)에서 사람이 읽는 슬러그(`/posts/새글-제목-입니다`)로 전환.

- **slugify**(`api/src/publishing/slugify.ts`): 한글 보존, 영문 소문자화, 공백→`-`, 허용 외 제거, 빈값→`post`, 최대 80자.
- **shared**: `PostSummaryDto`·`PostDetailDto`에 `slug` 추가.
- **api**: `Post.slug @unique` + 안전 마이그레이션(`20260605090000_add_post_slug`: nullable→`slug=id` 백필→NOT NULL→unique). 생성 시 유일 슬러그 부여(충돌 `-2/-3`). 공개 상세 `getPublishedDetail(idOrSlug)`가 **slug 우선·cuid 폴백**. 백필 스크립트 `backfill:slugs`(멱등, slug===id 인 행만).
- **web**: 목록/카드 링크 `/posts/{slug}`, 라우트 `/posts/:slug`. `usePost`가 한글 슬러그 인코딩. `PostDetail`이 응답 `slug`≠param이면 `/posts/{slug}`로 `replace`(cuid→canonical).

## Prisma 7 관련 결정 (중요 — 다음 태스크 영향)

Prisma 7의 새 `prisma-client` 생성기는 기본 ESM + 드라이버 어댑터 + wasm 엔진이라
NestJS(CJS)/Jest 환경에 맞추기 위해 다음을 조정함:

1. **생성기 CJS 출력**: `schema.prisma` generator에 `runtime = "nodejs"`, `moduleFormat = "cjs"` 추가.
   (데이터 모델·마이그레이션 불변. generated/는 .gitignore 처리, 커밋 제외.)
2. **드라이버 어댑터 필수**: `@prisma/adapter-pg` + `pg` 추가. PrismaService가
   `new PrismaPg({ connectionString: process.env.DATABASE_URL })`로 연결.
   → URL은 `datasources`/`datasourceUrl` 옵션이 아니라 **어댑터**로 주입.
3. **Jest wasm 엔진**: 엔진이 wasm 컴파일러를 동적 import → `--experimental-vm-modules` 필요.
   `cross-env`로 test 스크립트에 `NODE_OPTIONS=--experimental-vm-modules` 적용.
4. **`.js` 임포트 해석**: 생성 클라이언트가 `./x.js`로 import → jest `moduleNameMapper`
   `^(\\.{1,2}/.*)\\.js$ → $1` 추가.

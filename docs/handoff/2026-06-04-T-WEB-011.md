# Handoff — 작성자 이름 표시 (T-PUB-104 / T-WEB-011)

날짜: 2026-06-04
기능: author-display (글 목록·상세에 작성자 이름 노출)

## 무엇을 했나

글 목록과 글 상세에 **작성자 이름**이 표시되지 않던 문제를 구현으로 해결했다.
`User` 모델에 이름 필드가 없어, 표시 이름의 출처를 새로 정의했다(ADR-0017).

### 결정 (ADR-0017)
- `User`에 **`name`(NOT NULL, 표시 이름)** 추가. 공개 응답엔 `email` 대신 이 값을 노출(이메일 비공개).
- `Post`는 `author` 관계로 `name`을 읽음(비정규화 컬럼 없음 — 이름 변경 자동 반영).
- 작성자 출처 후보 중 "User.name 추가"를 선택(email 전체/로컬파트 노출 대안 기각).

## 변경 파일
- `docs/adr/0017-user-display-name-for-author.md` (신규, Accepted)

- `packages/api/prisma/migrations/20260604000000_add_user_display_name/` — nullable 추가 → email 로컬파트 백필 → NOT NULL
- `packages/api/src/auth/seed-operator.ts` — `name?` 파라미터(미지정 시 `email.split('@')[0]`)
- `packages/api/prisma/seed.ts` — `OPERATOR_NAME` 환경변수(선택) 전달
- `packages/api/src/publishing/post.service.ts` — 조회에 `author: { select: { name }}` 포함, `toSummary`/`toDetail`에 `authorName` 매핑
- `packages/shared/src/dto/post.ts` — `PostSummaryDto`/`PostDetailDto`에 `authorName: string`
- `packages/web/src/components/PostListView.tsx` — 카드 메타에 작성자 이름
- `packages/web/src/pages/PostDetail.tsx` — 상세 헤더 메타에 작성자 이름
- 테스트(RED→GREEN): post.service.spec, post.e2e-spec, seed-operator.spec, web PostList/PostDetail.test

## 운영 주의 / 후속

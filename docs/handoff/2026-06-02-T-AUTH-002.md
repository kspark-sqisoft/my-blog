## 다음 태스크가 알아야 할 것 (T-AUTH-003/004)

- **JwtService 설정은 AuthModule에서**: T-AUTH-002 spec은 `JwtModule.register({ secret: process.env.JWT_SECRET, signOptions: { expiresIn: '1h' } })`로 테스트했음. AuthModule도 동일 패턴(`JwtModule.registerAsync`로 env 주입) 권장.
- JWT payload는 `{ sub: userId, email }` → JwtStrategy(T-AUTH-003)는 `req.user = { id: payload.sub, email: payload.email }` 매핑.
- 토큰은 **httpOnly 쿠키 `access_token`**으로 내려야 함(ADR-0001) — 컨트롤러(T-AUTH-004)에서 `res.cookie(...)`.
- `JWT_SECRET` 환경변수 필요(.env.example에 키 존재). 테스트는 미설정 시 `test-secret` 기본.
- 반환 타입 `LoginResult { accessToken, user }`를 컨트롤러에서 사용.

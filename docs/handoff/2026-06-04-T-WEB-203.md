## 결정한 것
- 비디오 카드는 **첫 프레임만** 표시하고 인라인 재생은 막는다(`controls` 없음). 클릭은 항상 상세 이동.
- 첫 프레임 추출은 클라이언트 브라우저의 `<video preload="metadata">` 기본 동작만 사용 — ffmpeg/서버 의존성 0(ADR-0020).
- 카드 자체에 `aria-label` 등 텍스트는 없음(이미지 케이스와 동등). 후속에 a11y 보강 가능.

## 알려진 이슈
- 목록에 비디오 N 개가 보이면 N 번의 metadata + 첫 프레임 다운로드가 발생한다(ADR-0020 의 인지된 트레이드오프). 모바일 데이터/대역폭 모니터링 후속.
- 일부 브라우저는 `preload="metadata"` 가 첫 프레임을 충분히 못 보여주는 경우가 있다 — 후속에 `#t=0.1` fragment 등으로 강제 가능.

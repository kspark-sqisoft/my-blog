# Handoff — T-WEB-202 글 에디터 미디어 업로드 UX (이미지+비디오 통합)

날짜: 2026-06-04 / feature: video-upload / 근거: ADR-0020

## 한 일
글 에디터의 업로드 패널을 "이미지 전용"에서 "미디어(이미지+MP4 비디오) 통합"으로 확장했다.
서버는 이미 T-PUB-202 에서 통합 수락 중. 작성자 UX 만 일치시키는 변경.

- `PostEditor.tsx`:
  - 사이드 패널 제목 "커버 이미지" → "미디어", 힌트 "본문에 마크다운 이미지로 삽입됩니다" → "이미지 또는 MP4 비디오. 본문 마크다운에 한 줄로 삽입됩니다".
  - input `id="image" aria-label="이미지 업로드" accept="image/*"` → `id="media" aria-label="미디어 업로드" accept="image/*,video/mp4"`.
  - `ALLOWED_UPLOAD_MIME` 모듈 상수(api 의 `ALLOWED_MIME` 과 한 쌍): jpeg/png/gif/webp + mp4. `handleUpload` 가 `file.type` 으로 한 번 더 검증해 비허용은 setError + early return.
  - 본문 삽입 마크다운은 이미지/비디오 동일하게 `\n![file.name](url)`. 렌더러가 url 확장자로 `<img>`/`<video>` 자동 분기(T-WEB-201).
- `PostEditor.test.tsx`:
  - 기존 "이미지 업로드" 라벨 케이스를 "미디어 업로드" 라벨로 업데이트 + 응답 mock 에 `type: 'image'` 보강.
  - 4 신규 케이스: accept 속성, MP4 ![alt](url) 삽입, PDF 클라이언트 차단, MOV(video/quicktime) 클라이언트 차단.

## 결정한 것
- 클라이언트 화이트리스트는 **명시 5종**(jpeg/png/gif/webp/mp4) — accept 속성의 `image/*` 와이드카드는 OS 다이얼로그 편의용이고 강제력 없음. 실제 차단은 `file.type` 정확 매칭.
- 마크다운 삽입 형태는 이미지/비디오 동일(`![alt](url)`) — 렌더러가 분기를 책임지므로 작성자 UX 가 단순해진다.
- 변수/훅 이름 `useUploadImage` 는 그대로 유지(영향 범위 보존). 의미상 `useUploadMedia` 로 이름 변경은 후속 리팩터.

## 알려진 이슈
- 업로드 input 의 `e.target.value = ''` 초기화는 같은 파일 재선택 시 onChange 가 다시 발화하도록 보장한다. 차단 후 사용자가 동일 파일을 다시 시도해도 정상.
- 화이트리스트가 api/web 두 곳에 중복 — 공유 위치(예: `packages/shared/upload-mime.ts`)로 추출은 후속(작업 작아 별건).

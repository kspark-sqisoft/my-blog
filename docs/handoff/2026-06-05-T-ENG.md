# Handoff — 참여(Engagement): 좋아요 + 조회수 (T-ENG-001~005)

## 무엇을 했나
글 참여 기능(C) 전체를 격리 worktree(`worktree-engagement`)에서 구현. 새 **Engagement Context**.

- **T-ENG-001** 스키마: `Like(postId,userId)`·`PostView(postId,visitorKey)` 복합 PK 모델 +
  `Post.viewCount/likeCount` 비정규화 카운터. 마이그레이션 `20260605071948_add_engagement_likes_views`
  (추가 전용 — ADD COLUMN/CREATE TABLE). dev(blog)·test(blog_test) 적용.
- **T-ENG-002** 좋아요 API: `POST/DELETE /api/posts/:id/like` (로그인 필수, 멱등 토글, likedByMe).
- **T-ENG-003** 조회수 API: `POST /api/posts/:id/view` (공개, 방문자키 30분 dedup, 원문 IP/UA 미저장 해시만).
- **T-ENG-004** 노출: `PostDetailDto`(viewCount/likeCount/likedByMe)·`PostSummaryDto`(viewCount/likeCount),
  `GET /api/posts/:idOrSlug`에 `OptionalJwtAuthGuard`로 likedByMe 계산.
- **T-ENG-005** 웹: `LikeButton`(낙관적 업데이트, 비로그인 401→/login) + 조회수(eye) + 마운트 시 view 핑.

ADR-0024 신설. glossary(좋아요/조회수/방문자 키)·bounded-contexts(Engagement)·CLAUDE.md 엔드포인트 표 갱신.

## 다음 사람을 위한 메모
- **worktree 작업**: 이 브랜치는 메인과 별개. dev Docker 스택은 메인 디렉터리 소스를 바인드 마운트하므로
  worktree 편집은 핫리로드로 안 보인다. TDD(jest/vitest)는 worktree에서 정상. **브라우저 수동 도그푸딩은
  B(읽기경험) 머지 후 메인 디렉터리에서** 하면 한 화면에서 같이 확인된다.
- 머지 시 `PostDetail.tsx`는 B 세션과 겹칠 수 있음(3-way 머지). `feature_list.json`·`docs/tasks`는 additive 충돌.

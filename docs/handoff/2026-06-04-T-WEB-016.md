# Handoff — T-WEB-016 (글 에디터 클라이언트 입력 검증)

날짜: 2026-06-04 / feature: author-by-default

## 한 일
글 작성/수정 에디터(`PostEditor`)에서 저장 시 **클라이언트 검증**을 추가했다. 제목/본문이 비면
서버 400 이 나기 전에 상단 알림 배너를 띄우고, 빈 필드로 **포커스 이동 + 깜빡임(blink)** 한다.

- 제목 필수(공백 trim 후 빈 값 차단, 백엔드 `@MinLength(1)` 와 정합) → POST/PATCH 미전송.
- 본문 필수(빈 값 차단). 둘 다 통과해야 저장 진행.
- 검증 실패 메시지는 에디터 상단 단일 알림(`role="alert"`, `.ab-form-error`)으로 일원화(기존 태그 에러도 여기로).
- 빈 필드 시각 피드백: `.invalid` 클래스 + `ab-field-blink` 키프레임(테두리/링 깜빡임), `ref.focus()`.
- 저장 페이로드의 title 은 trim 해서 전송.

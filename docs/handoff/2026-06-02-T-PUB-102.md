## 한 일

- `UploadController`(`/api/uploads`, 운영자 전용):
  - `@UseGuards(JwtAuthGuard)` + `FileInterceptor('file', { fileFilter })`(메모리 스토리지).
  - fileFilter: 이미지 MIME 화이트리스트(png/jpeg/gif/webp) 외 → BadRequest(400).
  - 핸들러: `file.size > UPLOAD_MAX_BYTES` → PayloadTooLargeException(413).
  - `StorageProvider.save()` 호출 → `{ url, contentType, size }`(UploadResultDto) 반환(201).
- PublishingModule: UploadController 등록 + `{ provide: StorageProvider, useClass: LocalStorageProvider }`.
- `@types/multer` devDep, `.env.example`에 UPLOAD_* 추가.

## 설계 메모

- 크기 상한은 **핸들러에서 env(UPLOAD_MAX_BYTES) 기준 검증**(요청 시점). multer 하드 limit은 두지 않음 — 운영자 전용(인증 가드) 엔드포인트라 DoS 위험 낮음. 필요 시 추후 multer limits + MulterError→413 필터 추가 가능.
- 운영 정적 서빙: nginx `/uploads/`(prod compose) + UPLOAD_URL_BASE=/uploads 정합. 로컬 저장 디렉터리는 컨테이너 영속 볼륨 필요(추후 배포 시).

# Handoff — T-WEB-201 마크다운 렌더러 .mp4 자동 <video> 분기

날짜: 2026-06-04 / feature: video-upload / 근거: ADR-0020

## 한 일
PostDetail 의 마크다운 렌더러(`<Markdown>`)가 이미지 노드의 URL 확장자가 `.mp4` 면
`<video controls preload="metadata" playsInline aria-label={alt}>` 로 치환, 그 외엔 기존
`<img>` 그대로. 작성자 UX(마크다운 `![alt](url)`) 는 변경 없음.

- `packages/web/src/components/Markdown.tsx`:
  - `VIDEO_EXTENSIONS` 정규(`.mp4` + 쿼리/해시 허용)로 분기.
  - `MediaNode({ src, alt })` 컴포넌트 — `.mp4` 는 `<video>`, 그 외는 `<img>`.
  - `ReactMarkdown` 의 `components={{ img: MediaNode }}` 로 노드 변환을 마크다운 이미지 syntax 한 곳에서만 수행.
- `Markdown.test.tsx` 6 신규 케이스:
  - `.mp4` <video> 분기 + 속성(controls/preload/playsInline)
  - autoplay 없음
  - .jpg/.png/.gif/.webp 는 `<img>` 유지
  - alt → aria-label 매핑
  - raw `<video>` 텍스트는 렌더 0개(우회 경로 없음)

## TDD 사이클 확인
- RED: 6 신규 케이스 작성 → 3 fail (`.mp4` <video>, autoplay 검사, aria-label).
- GREEN: `Markdown.tsx` 에 `MediaNode` + `components` 옵션 추가 → 8/8 통과.
- REFACTOR: 비디오 확장자 정규를 `\.mp4(?:\?|#|$)` 로 (쿼리스트링/해시 url 도 안전하게 감지).

## 결정한 것
- 분기는 **렌더 시점**(`react-markdown` components) 에서만 — 작성자 UX 변경 없음, sanitize 정책 변경 없음, raw `<video>` 우회 경로 없음.
- alt → `aria-label`. `<video>` 는 `alt` 속성이 없으므로 접근성 텍스트는 aria-label 로 노출.
- 비디오 확장자는 현재 `.mp4` 한 종류(ADR-0020). 확장 시 정규 추가.
- `<video>` 의 cross-origin/sandbox 정책은 기본값(=동일 출처) 그대로. 외부 비디오 임베드는 ADR 범위 외.

## 알려진 이슈
- 비디오 노드는 컨테이너 너비 100% 등 시각 스타일이 아직 없음 — index.css 후속 보강 가능(컨텐츠 폭 안에서 정렬). T-WEB-202/203 진행 중 함께 다듬을 수 있음.

## 다음 세션이 알아야 할 것
- 다음 태스크 **T-WEB-202**: 글 에디터 미디어 업로드 UX. `<input accept="image/*,video/mp4">`, 라벨 "미디어 업로드". 업로드 응답의 `type` 에 무관하게 본문에 `![alt](url)` 그대로 삽입. 비허용 MIME 은 클라이언트 단계에서 알림 + 차단.
- T-WEB-203: 목록 카드 비디오 첫 프레임 커버. `coverImageUrl` 의 확장자가 `.mp4` 면 `<video preload='metadata' muted playsInline>` (controls 없음, 카드 인라인 재생 X).

# Handoff — 사용자 프로필 + 아바타 (T-PROF-001~006, ADR-0025)

## 무엇을 했나
격리 worktree(`worktree-profile`)에서 프로필/아바타 기능 전체 구현.

- **T-PROF-001** `User.avatarUrl String?` + 마이그레이션(`add_user_avatar_url`, 추가 전용). `AuthUserDto.avatarUrl`
  (login/register/me/PATCH 공통) — `auth.service.issue`·`jwt.strategy.validate` 갱신.
- **T-PROF-002** `POST /api/profile/avatar`(ProfileModule): 로그인 전 역할, 이미지 전용, 2MB, StorageProvider 재사용 →`{url}`.
- **T-PROF-003** `PATCH /api/auth/me`: 본인 name(1~50)·avatarUrl 수정. avatarUrl 은 로컬 `/uploads` 또는 null 만(외부 URL 400).
- **T-PROF-004** `CommentDto`·`PostSummaryDto`·`PostDetailDto` 에 `authorAvatarUrl` — 작성자 join, 익명·미설정 null.
- **T-PROF-005** 웹: `Avatar`(이미지|이니셜) + NavBar 아바타·이메일→`/profile`(앱 로드 fetchMe) + `/profile` 편집 페이지.
- **T-PROF-006** 웹: 댓글(`CommentTree`)·글(`PostDetail`·`PostListView`) 작성자에 Avatar.

ADR-0025 신설. glossary(프로필/아바타)·bounded-contexts(Auth)·CLAUDE.md 엔드포인트·feature_list(E16)·tasks 동기화.

## 메모 / 함정 (이전 교훈 반영)
- **shared dist / prisma client 재생성**: shared 소스 변경 후 `pnpm --filter @blog/shared run build`,
  스키마 변경 후 `prisma generate` 필요(둘 다 stale 시 api 컴파일/테스트가 깬다). 하네스가 자동화하지만
  worktree 수동 작업 시 주의(이번에도 prisma client stale 로 46개 실패 → regenerate 로 해소).
- avatarUrl 은 **로컬 /uploads 경로만** 저장(외부 URL 거부 — XSS/추적 방지). v1 은 교체 시 이전 파일 미정리(고아 허용).
- 아바타 업로드는 운영자 `/api/uploads`(비디오 포함)와 분리된 전용 엔드포인트(이미지 전용·2MB).
- web 은 `@testing-library/user-event` 미설치 → 테스트는 `fireEvent` 사용.
- 브라우저 도그푸딩(아바타 업로드·표시)은 머지 후 메인 디렉터리 dev 스택에서.

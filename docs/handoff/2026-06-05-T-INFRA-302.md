# Handoff — T-INFRA-302 Prisma Post.contentHtml 컬럼 + 마이그레이션

날짜: 2026-06-05 / feature: rich-text-editor / 근거: ADR-0021

## 한 일
Post 본문 모델 전환을 위한 DB 컬럼 추가. 기존 `contentMarkdown` 은 그대로 두고 새 `contentHtml`
컬럼을 빈 문자열 기본값으로 추가했다. T-INFRA-303 이 일회성 마이그레이션 스크립트로 값 채움.

  - 기존 `contentMarkdown` 라인에 `@deprecated` 주석(T-INFRA-303 마이그레이션 후 후속 ADR 에서 drop).
- 마이그레이션 파일: `prisma/migrations/20260604151420_add_post_content_html/migration.sql`.
- DB 양쪽 적용:
  - dev `blog`: `prisma migrate dev --name add_post_content_html` (마이그레이션 생성 + 적용)
  - 테스트 `blog_test`: `prisma migrate deploy` (적용만)

## 결정한 것
- 컬럼명은 Prisma 의 camelCase 기본을 그대로 — DB 에도 `"contentHtml"` 로 quoted identifier(다른 컬럼들도 동일 컨벤션).
- default `''` (NOT NULL) — null 표현으로 "본문 없음" 을 다루지 않고, 빈 문자열 = "아직 마이그레이션 안 됨 또는 빈 본문" 으로 단일 의미.
- 마이그레이션 시점: Prisma CLI 가 dev/test DB 모두에 그대로 적용 가능(타임스탬프 파일 1개).

## 알려진 이슈
- 기존 row 가 모두 `contentHtml=''` 상태 — T-INFRA-303 실행 전까지는 본문이 채워진 contentMarkdown 만 가지고 있는 셈. PostDetail 응답에 `contentHtml` 가 optional 이라 클라이언트 영향 없음.
- contentMarkdown 컬럼은 그대로 — 후속 ADR + 마이그레이션에서 제거(롤백 안전망).

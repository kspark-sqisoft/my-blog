---

새 세션 준비. 아래를 순서대로 실행하고, 하나라도 실패하면 즉시 멈추고 보고하라.

1. **환경 점검**: `bash init.sh` 실행.
   - 비0 종료(실패)면 출력을 보여주고 **멈춤** (다음 단계 진행 금지).

2. **직전 컨텍스트**: `docs/handoff/` 에서 가장 최신 파일 1개를 읽어 핵심만 요약.

3. **최근 변경**: `git log --oneline -10`.

4. **다음 태스크 결정**: `feature_list.json` 을 읽어 (사람용 .md 가 아니라 JSON 기준):
   - `status="todo"` 이고
   - `deps` 가 모두 `status="done"` 이며
   - `priority` 가 가장 작은(높은) 태스크 1개를 고른다.
   - 후보가 없으면 그 사실을 보고.

5. **확인 요청 후 멈춤**: 결정한 태스크의 **ID + title + acceptance 배열**만 보여주고 멈춘다.
   사용자가 "진행" 류로 확인하기 전에는 코드를 만들지 않는다.

확인 후 진행 시에는 `/implement {ID}` 의 TDD 절차(Red→Green→Refactor)를 따른다.
완료 후에는 직접 status 를 고치지 말고 `/finish {ID}` 로 마감한다.
